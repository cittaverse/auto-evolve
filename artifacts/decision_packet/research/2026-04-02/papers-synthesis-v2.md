## 核心关联主题

### 主题 1: 长期记忆架构 (Long-Term Memory Architecture)

**关联论文**: OmniMem, PsychAgent, Brainstacks, YC-Bench

**共同问题**: 如何在长程交互中维护信息的完整性和可检索性？

**技术对比**:
| 论文 | 记忆类型 | 组织方式 | 检索机制 | 更新策略 |
|------|----------|----------|----------|----------|
| OmniMem | 多模态 | 自主发现 | 研究引导 | 自动优化 |
| PsychAgent | 会话经验 | 线索图 | 规划引擎 | 技能演化 |
| Brainstacks | 认知原语 | Adapter Stack | 组合激活 | 零遗忘投影 |
| YC-Bench | 任务状态 | Scratchpad | 结构化读写 | 显式更新 |

**融合机会**:
```
┌─────────────────────────────────────────────────────────┐
│            CittaVerse 融合记忆架构 (提议)               │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  Layer 1: Scratchpad (YC-Bench)                        │
│  - 结构化用户画像、会话状态、安全标记                   │
│  - 快速读写，低延迟                                     │
│                                                         │
│  Layer 2: Memory Anchors (PsychAgent)                  │
│  - 回忆线索图、事件索引、情感强度                       │
│  - 支持跨会话连续性                                     │
│                                                         │
│  Layer 3: Cognitive Stacks (Brainstacks)               │
│  - 用户特异性认知原语组合                               │
│  - 零遗忘持续学习                                       │
│                                                         │
│  Layer 4: Autoresearch Loop (OmniMem)                  │
│  - 定期诊断记忆检索失败                                 │
│  - 自动优化组织方式和检索策略                           │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**技术趋势**: 从"单一记忆存储" → "分层记忆架构" → "自主优化记忆"

### 主题 2: 个性化适配 (Personalization)

**关联论文**: VRF, PsychAgent, Narrative Fingerprints

**共同问题**: 如何快速适配新用户并持续优化个性化策略？

**技术对比**:
| 论文 | 表示方式 | 学习信号 | 少样本能力 | 不确定性 |
|------|----------|----------|------------|----------|
| VRF | 变分分布 | 偏好反馈 | ✅ 强 | ✅ 显式 |
| PsychAgent | 技能演化 | 会话结果 | 🟡 中 | ❌ 隐式 |
| Narrative Fingerprints | 新颖性曲线 | 文本风格 | ❌ 弱 | ❌ 无 |

**融合机会**:
```
┌─────────────────────────────────────────────────────────┐
│          CittaVerse 个性化适配框架 (提议)               │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  阶段 1: 冷启动 (VRF)                                   │
│  - 变分编码器从少量交互推断用户偏好分布                 │
│  - 输出：偏好 + 不确定性                                │

│                                                         │
└─────────────────────────────────────────────────────────┘
```

**技术趋势**: 从"静态用户画像" → "动态偏好分布" → "演化技能库"

## 可借鉴的技术模块

### 模块 1: Scratchpad 长程记忆 (优先级: P0)

**来源**: YC-Bench (2604.01212)  
**成熟度**: ✅ 概念清晰，实现简单  
**依赖**: 无  
**预期收益**: 跨会话连贯性 +50%

**实现要点**:
- 结构化存储：用户画像、回忆锚点、会话状态、演化记录
- 更新策略：会话后批量更新，避免实时写入错误
- 衰减机制：旧信息按时间衰减，避免永久保留过时信息

---

### 模块 2: 变分用户偏好编码器 (优先级: P1)

**来源**: VRF (2604.00997)  
**成熟度**: 🟡 需要训练数据  
**依赖**: 用户反馈数据  
**预期收益**: 新用户冷启动质量 +45%

**实现要点**:
- 共享偏好基：从现有用户聚类得到 K 个原型
- 变分编码器：输出用户偏好分布 (μ, σ²)
- 不确定性应用：高不确定性时采用探索策略

---

### 模块 3: 零遗忘 Adapter Stack (优先级: P1)

**来源**: Brainstacks (2604.01152)  
**成熟度**: 🟡 需要预训练 infrastructure  
**依赖**: LoRA 训练框架  
**预期收益**: 持续学习无遗忘

**实现要点**:
- 预定义认知原语 stack (语言、推理、记忆、情感...)
- Null-space projection 确保新 stack 不干扰旧能力
- Stack 选择器根据任务动态组合

---

### 模块 4: 自主研究诊断 Loop (优先级: P2)

**来源**: OmniMem (2604.01007)  
**成熟度**: 🟡 需要实验框架  
**依赖**: 标注数据、实验追踪  
**预期收益**: 减少手动调参 70%

**实现要点**:
- 设计空间定义：检索策略、组织方式、评分函数
- 自动实验：在历史数据上评估配置
- 诊断修复：分析失败案例，提出架构修改

---

### 模块 5: 状态化用户模拟器 (优先级: P2)

**来源**: PARE (2604.00842)  
**成熟度**: 🟡 需要 FSM 建模  
**依赖**: 用户行为数据  
**预期收益**: 离线评估能力

**实现要点**:
- 将用户回忆探索建模为 FSM
- 模拟用户具有状态化行为 (疲劳、改变主意、犯错)
- 评估干预时机、目标推断准确率
