## 二、技术趋势识别

### 趋势 1: 从静态架构 → 自主演化系统

**证据链**:
| 论文 | 演化机制 | 自主程度 |
|------|----------|----------|
| OmniMem | Autoresearch pipeline 自动探索记忆设计空间 | 🔴 高 |
| PsychAgent | Skill Evolution Engine 从历史轨迹提取技能 | 🟡 中 |
| HERA | Role-Aware Prompt Evolution 动态优化 Agent 行为 | 🟡 中 |
| Brainstacks | 外循环训练新 stack，内循环残差增强 | 🟡 中 |

**趋势解读**:
- 2026 Q1 的研究明显从"设计更好的架构"转向"设计能自我改进的架构"
- OmniMem 的 +411% 提升中，超参调优贡献最小，架构发现和 bug 修复贡献最大
- 这暗示手动设计记忆系统已接近瓶颈，自动发现是下一步

**CittaVerse 含义**:
阿宝的记忆系统不应是静态设计的，而应具备：
1. 自动诊断记忆检索失败模式的能力
2. 自动尝试不同记忆组织策略的能力
3. 自动评估哪种策略对特定用户最有效的能力

---

### 趋势 2: 从单 Agent → 动态多 Agent 协作

**证据链**:
| 论文 | 协作模式 | 动态性 |
|------|----------|--------|
| HERA | 全局优化 Agent 拓扑 + 局部优化角色提示 | 🔴 高 |
| PARE | 多应用状态机编排，模拟用户跨应用行为 | 🟡 中 |
| CARE | 远程 LLM(类别生成) + 本地 LLM(决策) | 🟢 低 |

**趋势解读**:
- HERA 的 38.69% 平均提升来自两个层面：全局编排 + 局部行为
- 拓扑分析显示"稀疏探索产生紧凑、高利用率网络"——暗示最优协作结构是涌现的，非预设的
- PARE 将应用建模为状态机，捕捉交互的顺序性和状态依赖性

**CittaVerse 含义**:
回忆引导可能涉及多角色协作：
- **引导 Agent**: 发起回忆话题，温和追问
- **评估 Agent**: 监测情绪安全，防止深挖创伤
- **叙事 Agent**: 整理回忆素材成故事
- **锚点 Agent**: 关联个人记忆与集体记忆

HERA 的动态编排可优化这些角色何时激活、如何协作

### 趋势 3: 从短期对话 → 长程连贯性

**证据链**:
| 论文 | 长程机制 | 跨度 |
|------|----------|------|
| YC-Bench | Scratchpad 持久化跨上下文截断信息 | 数百轮/一年 |
| PsychAgent | Memory-Augmented Planning Engine | 多会话 |
| OmniMem | Lifelong Multimodal Memory | 长期 |
| Brainstacks | Frozen Stack 累积领域能力 | 持续 |

**趋势解读**:
- YC-Bench 发现 Scratchpad 是成功的最强预测因子
- 47% 破产源于"对抗性客户检测失败"——暗示长程中维持情境意识的困难
- GLM-5 以 11× 更低成本达到接近 Claude Opus 4.6 的性能——成本效率重要

**CittaVerse 含义**:
阿宝与用户互动是数月/数年的长程关系：
1. 需要 Scratchpad 机制维护跨会话关键信息（用户偏好、禁忌话题、重要回忆锚点）
2. 需要检测"对抗性"情境（如用户情绪恶化、回忆冲突）
3. 需要在成本可控下维持长程连贯性

### 趋势 4: 从确定性 → 不确定性感知

**证据链**:
| 论文 | 不确定性建模 | 应用场景 |
|------|--------------|----------|
| VRF | 变分分布表示用户偏好，方差衰减损失 | 个性化 |
| ORCA | Conformal prediction 提供置信估计 | 推理校准 |
| CARE | 证据冲突检测，多阶段推理 | 医疗决策 |

**趋势解读**:
- VRF 将用户偏好从点估计升级为分布估计，在少样本和未见用户上优势明显
- ORCA 在分布外设置下将效率从 24.8% 提升至 67.0%，同时维持低错误率
- CARE 专门处理证据内部不一致场景（症状 vs 体征矛盾）

**CittaVerse 含义**:
阿宝需要不确定性感知：
1. 对用户叙事偏好的估计应是分布而非点（用户可能自己也说不清偏好）
2. 对回忆引导效果的评估应有置信区间（避免过度自信的错误引导）
3. 对用户回忆内容的冲突应有检测和处理机制（时间线矛盾、事实冲突）

### 趋势 5: 从通用能力 → 可迁移认知原语

**证据链**:
| 论文 | 核心发现 |
|------|----------|
| Brainstacks | Domain stack 编码可迁移认知原语（指令跟随、数值推理、程序逻辑、CoT 结构）而非领域知识 |
| Narrative Fingerprints | 作者指纹在多尺度上可识别，部分独立于体裁 |

**趋势解读**:
- Brainstacks 的 router 发现医疗提示 97% 路由到 chat+math stack，尽管这些 stack 零医疗数据
- 这暗示"领域能力"实际是底层认知原语的组合，而非领域专用知识
- Narrative Fingerprints 发现古典与现代作者指纹强度相当，暗示叙事风格有跨时代稳定性

**CittaVerse 含义**:
阿宝的个性化可能不需要为每个用户训练专用模型：
1. 识别用户需要的认知原语组合（如"需要温和引导"、"需要具体追问"、"需要抽象总结"）
2. 动态路由到对应的 stack/模块
3. 用户间可共享认知原语，仅权重不同

## 四、技术空白与机会

### 空白 1: 老年特定记忆模型

**现状**: 10 篇论文中无一篇专门针对老年人记忆特征
- PsychAgent 是心理咨询，非老年回忆
- 老年记忆特点：自传体记忆保留较好、工作记忆下降、情感调节需求高

**机会**: 为 CittaVerse 建立老年记忆专用模型可能是差异化优势

### 空白 2: 回忆质量评估基准

**现状**: 
- OmniMem 使用 LoCoMo、Mem-Gallery（通用记忆基准）
- YC-Bench 是创业模拟基准
- 无回忆引导质量评估基准

**机会**: 建立"回忆引导质量评估基准"可成为 CittaVerse 的学术贡献

### 空白 3: 情感安全与回忆深挖的平衡

**现状**:
- PsychAgent 是心理咨询，有专业伦理约束
- CARE 处理医疗证据冲突，非情感安全
- 无专门研究"回忆引导中何时停止深挖"的工作

**机会**: CittaVerse 的 emotional-safety skill 可发展为学术研究贡献
