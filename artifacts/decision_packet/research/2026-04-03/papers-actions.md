## 一、高优先级行动 (P0 — 1-2 周内启动)

### P0-1: 实现分层记忆架构 (HMO 启发)

**来源**: arXiv:2604.01670

**行动内容**:
```
当前状态 → 目标状态
单层对话历史 → 三层记忆结构:
├── L1 工作记忆 (最近 3 轮对话，100% 保留)
├── L2 情景记忆 (最近 7 天会话，关键词 + 情感标签)
└── L3 自传记忆 (人生故事书，结构化叙事)
```

**技术要点**:
- 用户画像驱动 L2→L1 提升 (频繁访问的记忆升级)
- L3 采用叙事结构 (时间线/主题/人物关系图谱)

**预期收益**:
- 对话连贯性提升 (减少"您之前说过..."类错误)
- 个性化程度提高 (记忆访问模式反映用户特点)

**负责人**: Core (工程实现)  
**Hulk 支持**: 提供 HMO 论文详细解读 + 架构设计建议

### P0-2: 实现自适应记忆遗忘 (Budget-Aware Forgetting)

**来源**: arXiv:2604.02280, arXiv:2604.01664

**行动内容**:
```python
# 记忆保留评分公式 (建议)
retention_score = (
    0.4 * recency_decay(days_since_event) +
    0.3 * frequency_log(access_count) +
    0.3 * semantic_alignment(emotional_weight, user_profile)
)

# 遗忘策略
if retention_score < threshold:
    compress_to_summary()  # 非删除，而是压缩
if retention_score < threshold * 0.5:
    archive_to_L3()  # 移至自传记忆层
```

**技术要点**:
- 情感权重 (emotional_weight) 需结合老年人特点
- 压缩策略：保留核心事实 + 情感基调，删除细节

**预期收益**:
- 上下文使用效率提升 30-50%
- 减少虚假记忆传播 (论文报告 6.8%→更低)

**负责人**: Core  
**Hulk 支持**: 提供评分公式调优建议

### P0-3: 构建"叙事质量评估"内部基准

**来源**: arXiv:2604.01966 (MyEgo 方法论启发)

**行动内容**:
```
基准设计:
├── 数据收集: 50-100 位老年人真实对话 (脱敏)
├── 评估维度:
│   ├── 内部细节 (具体事件、时间、地点)
│   ├── 外部细节 (情感、反思、意义)
│   ├── 事件分段 (清晰的时间/主题边界)
│   └── 叙事连贯性 (因果关系、主题一致性)
└── 评分标准: 人工标注 + LLM 自动评分一致性验证
```

**技术要点**:
- 参考 MyEgo 的 5K 问题设计方法
- 需考虑中国文化背景 (集体主义叙事 vs 个人主义)

**预期收益**:
- 量化产品效果 (当前缺乏客观指标)
- 支撑论文/专利产出 (方法论创新)

**负责人**: Hulk (研究设计) + Core (数据管道)  
**时间线**: 2 周完成初版

## 二、中优先级行动 (P1 — 1 个月内启动)

### P1-1: 用户画像建模优化

**来源**: arXiv:2604.01670 (HMO 的用户画像驱动)

**行动内容**:
```
用户画像维度:
├── 认知特征: 记忆清晰度、叙事详细度、时间定向力
├── 情感特征: 正向/负向回忆偏好、情感表达强度
├── 社交特征: 家庭关系网络、重要他人提及频率
└── 文化特征: 代际语言习惯、历史事件参照点
```

**技术要点**:
- 冷启动问题：首次对话后生成初始画像
- 动态更新：每次对话后微调

**预期收益**:
- 记忆重分布更精准
- 回忆触发更符合个人特点

---

### P1-2: 元认知监控模块

**来源**: arXiv:2604.02318, arXiv:2604.01951

**行动内容**:
```
监控能力:
├── 记忆模糊检测: 识别用户回答中的不确定性信号
├── 叙事断裂检测: 识别时间线/逻辑矛盾
├── 情感波动检测: 识别回忆引发的情绪变化
└── 主动干预: "您刚才提到 XX，想多说说吗？"
```

**技术要点**:
- 惊喜阈值 (surprisal threshold) 需调优
- 干预频率需克制 (避免打扰感)

---

### P1-3: 多 Agent 叙事协作架构

**来源**: arXiv:2604.02211 (MAVRS 启发)

**行动内容**:
```
Agent 角色设计:
├── 倾听者 Agent: 专注理解当前叙述
├── 记忆者 Agent: 负责存储和检索相关记忆
├── 连接者 Agent: 发现跨时间/主题关联
├── 引导者 Agent: 生成温和的回忆提示
└── 整合者 Agent: 生成人生故事书章节
```

**技术要点**:
- 参考 MAVRS 的 Agent 分工模式
- 需设计 Agent 间通信协议
