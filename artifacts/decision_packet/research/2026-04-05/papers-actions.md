## 二、P0 行动 (立即执行)

### P0-1: 实现分层记忆架构

**来源**: TraceMem (三阶段管道) + Sophia (元层)

**行动内容**:
```
短期记忆层 (会话级)
├── 话题分割 (演绎式)
├── 事件边界检测
└── 语义表示提取

情景记忆层 (用户级)
├── 事件摘要
├── 用户轨迹蒸馏
└── 时间戳标记

语义记忆层 (知识库级)
├── 层次聚类
├── 主题线程组织
└── 跨会话关联
```

**技术选型**:
- 话题分割：基于嵌入相似度 + 启发式边界
- 事件提取：LLM + 预定义 schema
- 聚类：HDBSCAN 或层次聚类

**验收标准**:
- [ ] 支持跨 10+ 会话的记忆检索
- [ ] 话题分割准确率 > 80% (人工评估)
- [ ] 检索延迟 < 500ms

**负责人**: Core (工程实现)  
**依赖**: 无  
**风险**: 话题分割质量不稳定 → 缓解：人工校准初始版本

---

### P0-2: 实现人格引导叙事

**来源**: TraceMem (用户特定轨迹) + Role-Playing (Persona)

**行动内容**:
1. **用户 Persona 建模**:
   - 基础信息 (年龄、职业、地域)
   - 叙事风格偏好 (详细/简洁、情感/事实)
   - 记忆主题偏好 (家庭/工作/爱好)

2. **人格引导的记忆检索**:
   - 检索时加权用户偏好主题
   - 叙事生成时匹配用户风格

3. **Persona 演化追踪**:
   - 记录用户叙事风格变化
   - 检测认知状态变化信号

**技术选型**:
- Persona 嵌入：用户 profile → 向量
- 风格匹配：Few-shot prompting + 风格标签

**验收标准**:
- [ ] 用户可感知叙事个性化
- [ ] Persona 更新机制正常工作
- [ ] 支持多用户隔离

**负责人**: Core + Midas (产品设计)  
**依赖**: P0-1 分层记忆架构  
**风险**: 个性化可能强化偏见 → 缓解：定期审核

---

### P0-3: 构建叙事质量评估基准

**来源**: EvolMem (认知驱动基准) + NCT (叙事连续性测试)

**行动内容**:
1. **评估维度设计**:
   - 准确性 (事实一致性)
   - 连贯性 (时间/因果逻辑)
   - 丰富性 (内部/外部细节比例)
   - 情感表达 (情感词汇密度)
   - 连续性 (跨会话身份一致性)

2. **测试集构建**:
   - 收集 50+ 真实老年叙事样本
   - 人工标注质量分数
   - 构建自动评估脚本

3. **基线建立**:
   - 评估当前 CittaVerse 表现
   - 对比通用 LLM (GPT-4, Claude)
   - 设定改进目标

**验收标准**:
- [ ] 评估维度覆盖核心质量要素
- [ ] 测试集具有代表性
- [ ] 自动评估与人工评分相关性 > 0.7

**负责人**: Hulk (研究) + Core (工程)  
**依赖**: 无  
**风险**: 人工标注成本高 → 缓解：先小样本验证
