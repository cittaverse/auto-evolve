## 1. TraceMem: Weaving Narrative Memory Schemata from User Conversational Traces

**arXiv**: 2602.09712 [cs.CL] | **日期**: 2026-02-10

### 核心问题
LLM 的长期交互瓶颈：有限的上下文窗口无法管理随时间延长的对话历史。现有记忆系统将交互视为碎片化片段，未能捕捉对话流的叙事连贯性。

### 方法创新
**三阶段认知启发管道**:
1. **短期记忆处理**: 演绎式话题分割，划定事件边界并提取语义表示
2. **突触记忆巩固**: 将事件摘要为情景记忆，再与语义一起蒸馏为用户特定轨迹
3. **系统记忆巩固**: 两阶段层次聚类，将轨迹组织成统一主题下的连贯、时变叙事线程

**输出**: 结构化用户记忆卡 → 形成叙事记忆图式  
**检索**: 提供代理搜索机制增强推理过程

### 实验结果
- **基准**: LoCoMo benchmark
- **表现**: SOTA 性能
- **优势**: 在多跳推理和时间推理上超越基线

### 局限性
- 依赖话题分割质量
- 层次聚类可能丢失细粒度信息
- 代码已开源但需验证跨领域泛化

### 对 CittaVerse 的启示
⭐⭐⭐ **高度可借鉴**: 叙事记忆图式与"人生故事书"产品形态高度契合

## 2. ProcMEM: Learning Reusable Procedural Memory from Experience via Non-Parametric PPO

**arXiv**: 2602.01869 [cs.AI] | **日期**: 2026-02-02

### 核心问题
LLM Agent 在顺序决策中依赖即时推理，即使在重复场景中也重新推导解决方案，导致计算冗余和执行不稳定。

### 方法创新
**Skill-MDP 形式化**: 将被动情景叙事转换为可执行技能，定义为：
- 激活条件
- 执行条件
- 终止条件

**Non-Parametric PPO**: 
- 利用语义梯度生成高质量候选
- PPO Gate 进行稳健的技能验证
- 基于分数的维护保持紧凑、高质量的程序性记忆

### 实验结果
- **场景**: 域内、跨任务、跨代理
- **成果**: 优越的复用率、显著性能提升、极致记忆压缩
- **可视化**: 技能进化轨迹和分布透明展示知识积累

### 局限性
- 需要足够的交互经验积累
- 技能验证门限需要调优

### 对 CittaVerse 的启示
⭐⭐ **中度借鉴**: 程序性记忆可用于引导老年人叙事流程的标准化

## 3. Structured Episodic Event Memory

**arXiv**: 2601.06411 [cs.CL/cs.AI] | **日期**: 2026-01-09

### 核心问题
当前 LLM 记忆方法缺乏对情景事件的结构化组织，难以支持复杂的时间推理和事件关联。

### 方法创新
**结构化情景事件记忆架构**:
- 事件边界检测
- 事件要素提取 (参与者、地点、时间、动作)
- 事件间关系建模 (因果、时序、层级)

### 实验结果
- 在事件问答和时间推理任务上表现优异
- 支持多事件关联查询

### 局限性
- 摘要信息有限，需查看完整论文
- 事件检测依赖预定义 schema

### 对 CittaVerse 的启示
⭐⭐⭐ **高度可借鉴**: 事件结构化是叙事质量评估的基础

## 4. TeleMem: Building Long-Term and Multimodal Memory for Agentic AI

**arXiv**: 2601.06037 [cs.CL/cs.AI/cs.CV] | **日期**: 2026-01-12

### 核心问题
LLM 难以维持长期交互，RAG 缺乏可靠的存储更新和细化机制。多模态记忆整合是开放挑战。

### 方法创新
**长期多模态记忆系统**:
- 支持文本、图像、语音等多模态输入
- 动态记忆更新和细化
- 跨模态关联检索

### 实验结果
- 在长对话和多模态 QA 任务上验证
- 记忆更新机制有效防止信息过时

### 局限性

- 计算开销较大

### 对 CittaVerse 的启示
⭐⭐ **中度借鉴**: 多模态记忆支持语音 + 文本的混合叙事输入
