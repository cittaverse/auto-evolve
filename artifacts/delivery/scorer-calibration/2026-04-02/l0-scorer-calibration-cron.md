### V1: 情感词堆砌漏洞

**现象**: 纯情感词列表可得 4.3 分 (高唤醒)，无叙事结构仍可获高分。

**根因**: 情绪词汇计数无上限，未考虑词汇多样性和上下文合理性。

**修复方案 (v0.6)**:
```python
def check_emotion_diversity(emotion_words: List[str]) -> float:
    unique_ratio = len(set(emotion_words)) / len(emotion_words)
    counter = Counter(emotion_words)
    max_repeat = max(counter.values())
    repeat_penalty = 1.0 / (max_repeat / 3) if max_repeat > 3 else 1.0
    return unique_ratio * repeat_penalty

# 应用惩罚
score = score * (0.5 + 0.5 * diversity)
```

---

### V2: 关键词堆砌漏洞

**现象**: 精心构造的"情感词 + 身份词 + 完整事件"可得 100 分 S 级。

**根因**: 缺乏"叙事自然度"或"语义连贯性"检测。

**修复方案 (v0.6)**: 引入 LLM 仲裁层检测不自然叙事。

---

### V3: assign_grade 边界漏洞

**现象**: `assign_grade(-1) → D`, `assign_grade(101) → S`

**根因**: 未增加 clamp 或 assert 验证。

**修复方案 (v0.6)**:
```python
def assign_grade(score):
    assert 0 <= score <= 100, f"Invalid score: {score}"
    score = max(0, min(100, score))  # clamp
    ...
```

---

### V4: LLM 输出格式漂移

**现象**: LLM 返回非标准 JSON 时解析失败，fallback 到规则引擎。

**影响**: 安全性无风险，但评分质量可能下降。

**修复方案 (v0.7)**:
1. LLM prompt 中嵌入 JSON Schema 约束
2. 增加解析失败重试 (最多 2 次)
3. 增加 fallback 日志和监控指标
