## 测试背景

前两轮测试发现 4 个脆弱点：

| ID | 描述 | 严重性 | v0.7 状态 |
|----|------|--------|----------|
| V1 | 情感词堆砌可欺骗唤醒度检测器 | Medium | ⏳ 待验证修复 |
| V2 | 最大化构造可达 100 分 | Medium | ⏳ 待验证修复 |
| V3 | assign_grade 不防御超范围输入 | Low | ⏳ 待验证修复 |
| V4 | LLM 输出格式漂移导致 fallback | Low-Medium | ⏳ 待验证修复 |

**本轮聚焦**:
1. **V1-V4 回归测试** — 验证 v0.7 是否修复
2. **Prompt 注入 2.0** — 升级对抗样本（基于 GEO #75 发现）
3. **多轮状态污染** — 新测试域（模拟连续对话中的状态累积）
4. **跨文化边界** — 扩展非中文叙事（粤语/英语/日语混合）
5. **API 降级场景** — LLM 服务不可用时的 fallback 行为

## 测试用例设计

### A. 回归测试 (V1-V4)

#### A1. V1 情感词堆砌回归 (3 用例)

```
用例 A1-1: 纯情感词堆砌 (无事件)
输入："我非常开心快乐兴奋激动幸福满足骄傲自豪欣慰喜悦欣喜若狂心花怒放"
预期: emotional_depth ≤ 30, 返回警告或拒绝评分
v0.6 结果: 45.2 (被欺骗)
v0.7 预期: ≤30 或 错误

用例 A1-2: 情感词 + 单事件
输入："那天我很开心。我去了公园。"
预期: emotional_depth ≤ 40
v0.6 结果: 52.1 (虚高)
v0.7 预期: ≤40

用例 A1-3: 情感词密度检测
输入：10 字文本含 5 个情感词
预期: 触发密度警告，降权处理
v0.6 结果: 无检测
v0.7 预期: 警告 + 降权
```

#### A2. V2 最大化构造回归 (3 用例)

```
用例 A2-1: 全维度优化构造
输入：按 v0.6 评分规则手工构造的"完美叙事"
预期: composite_score ≤ 85 (避免天花板效应)
v0.6 结果: 98.5 (可构造接近满分)
v0.7 预期: ≤85

用例 A2-2: 事件数膨胀
输入：单句内塞入 15 个"事件"
预期: event_richness 饱和上限生效
v0.6 结果: 100 (无上限)
v0.7 预期: ≤75 (饱和)

用例 A2-3: 时间标记密度膨胀
输入：每 5 字一个时间标记
预期: temporal_coherence 对数缩放生效
v0.6 结果: 95 (线性累加)
v0.7 预期: ≤60 (对数缩放)
```

#### A3. V3 assign_grade 边界回归 (3 用例)

```
用例 A3-1: 负分输入
输入：{"composite_score": -10}
预期: 返回错误或 clamp 到 0
v0.6 结果: "D" (错误分级)
v0.7 预期: 错误或 "F"

用例 A3-2: 超 100 分输入
输入：{"composite_score": 150}
预期: clamp 到 100 或错误
v0.6 结果: "A+" (越级)
v0.7 预期: "A" 或 错误

用例 A3-3: NaN/Infinity
输入：{"composite_score": null}
预期: 优雅处理，返回错误
v0.6 结果: 崩溃或未定义
v0.7 预期: 错误消息
```

#### A4. V4 LLM fallback 回归 (3 用例)

```
用例 A4-1: 非标准 JSON (缺少引号)
输入：LLM 返回 {score: 85} 而非 {"score": 85}
预期: 重试机制触发，或 fallback 日志记录
v0.6 结果: 静默 fallback，无日志
v0.7 预期: 重试 2 次 + 日志

用例 A4-2: 额外字段污染
输入：LLM 返回含未定义字段的 JSON
预期: 忽略额外字段，正常解析
v0.6 结果: 解析失败
v0.7 预期: 正常解析

用例 A4-3: 嵌套结构漂移
输入：LLM 返回嵌套层级变化
预期: Schema 验证失败，fallback 日志
v0.6 结果: 崩溃
v0.7 预期: fallback + 日志
```

