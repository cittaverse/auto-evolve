# GEO 自主迭代协议 v1.0

> **GEO** = GitHub Engine Optimization
> 目标：通过自主迭代循环，持续提升 CittaVerse GitHub 仓库的索引质量、引用数和影响力

---

## 核心原则

1. **研究先行**：每次迭代前，先扫描竞品/趋势/缺口
2. **小步快跑**：30 分钟/轮，固定预算，避免过度优化
3. **单一指标**：每轮聚焦一个核心指标（索引页/引用/Star）
4. **闭环学习**：每轮沉淀一条可复用的经验

---

## 迭代循环

```
研究 (5min) → 规划 (5min) → 执行 (15min) → 验证 (3min) → 学习 (2min)
   ↑                                                              │
   └──────────────────────────────────────────────────────────────┘
```

### 阶段 1: 研究 (research.sh)

**目标**：发现优化机会

**动作**：
- 扫描竞品仓库（LLM-MCI-detection, LLMCARE, awesome-mental-health 等）
- 检查 Google/Bing 索引状态
- 识别内容缺口（缺失的关键词、未覆盖的场景）

**产出**：`research-notes.md`

---

### 阶段 2: 规划 (plan.sh)

**目标**：确定本轮执行事项

**动作**：
- 从研究笔记中提取 1-3 个高优先级任务
- 评估工作量（必须在 15min 内完成）
- 生成执行清单

**产出**：`plan.md`

---

### 阶段 3: 执行 (execute.sh)

**目标**：完成规划的任务

**动作**：
- 更新 README/文档
- 添加新内容（案例/资源/代码）
- 优化 SEO（标题/关键词/内部链接）

**产出**：Git commits

---

### 阶段 4: 验证 (verify.sh)

**目标**：确认执行质量

**动作**：
- 检查 CI 状态
- 验证链接有效性
- 确认内容符合规范

**产出**：`verification-report.md`

---

### 阶段 5: 学习 (learn.sh)

**目标**：沉淀经验

**动作**：
- 记录本轮成功/失败
- 更新协议或脚本
- 归档到 memory/

**产出**：`learning.md` → `memory/YYYY-MM-DD-geo-iteration-N.md`

---

## 核心指标

| 指标 | 目标 | 测量频率 |
|------|------|----------|
| 迭代完成率 | >90% | 每轮 |
| 平均迭代时间 | <30min | 每轮 |
| CI 通过率 | 100% | 每轮 |
| 索引页面增长 | +5 页/轮 | 每日 |
| GitHub Stars 增长 | +10/周 | 每周 |

---

## 使用方式

```bash
# 运行单轮迭代
./geo-loop.sh 1

# 持续运行 (每 6 小时一轮)
while true; do ./geo-loop.sh $((++ITERATION)); sleep 21600; done
```
