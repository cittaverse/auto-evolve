## Question

L0/VSNC 评分管道各组件的独立贡献是什么？是否存在过度设计的组件？简化系统是否能获得更好的性能？

## Bottom Line

**简化系统全面优于复杂系统：`llm_only` 和 `minimal` 配置比完整系统评分 +5.7 分、稳定性 3 倍提升。C1(情绪唤醒度)/C2(动态比例)/C6(投票加权) 可安全移除。**

## Key Findings

### 1. 简化系统全面胜出

| 配置 | 评分 | 稳定性 (std) | vs Full |
|------|------|-------------|---------|
| `full` | 56.76 | 4.22 | 基线 |
| `llm_only` | **62.48** | **1.38** | **+5.73 / 3.06x** |
| `minimal` | **61.56** | **1.36** | **+4.80 / 3.10x** |
| `rule_only` | **61.56** | **1.36** | **+4.80 / 3.10x** |
| `hybrid` | **61.56** | **1.36** | **+4.80 / 3.10x** |

**洞察**: 复杂系统组件间存在负向交互，简化后性能反而提升。

### 2. 组件影响排名

| 排名 | 组件 | 移除后评分变化 | 解读 |
|------|------|----------------|------|
| 1 | C4 (L0 六维) | **+5.61** | 六维评分有显著惩罚效应 |
| 2 | C7 (仲裁) | **-3.29** | 仲裁倾向于调高分数 |
| 3 | C3 (LLM 事件) | -1.72 | LLM 事件提取有小幅增益 |
| 4 | C5 (多 Agent) | -1.53 | 多 Agent 有小幅增益 |
| 5 | C1 (情绪唤醒度) | **+0.13** | **影响可忽略** |
| 6 | C2 (动态比例) | **±0.00** | **完全冗余** |
| 7 | C6 (投票加权) | **±0.00** | **完全冗余** |

### 3. 等级分布剧变

| 配置 | C 级 | D 级 | 解读 |
|------|-----|-----|------|
| `full` | 20 | 30 | 基线，分布分散 |
| `w/o_C4_simplified_l0` | 48 | 2 | 简化后几乎全 C |
| `llm_only` | 50 | 0 | **全部 C 级，最一致** |
| `minimal` | 50 | 0 | **全部 C 级，最一致** |

**洞察**: 简化系统不仅评分更高，而且等级判定更一致。

### 4. 仲裁率异常

| 配置 | 仲裁率 |
|------|--------|
| `full` | 100% |
| `w/o_C7_no_arbitration` | 0% |
| `w/o_C5_single_judge` | 0% |

**问题**: 100% 仲裁率意味着所有样本都触发仲裁，阈值设置可能过低或评委分歧系统性偏高。

## Confidence / Uncertainty

### 高置信度
- C1/C2/C6 影响微弱 → 可安全移除
- 简化系统稳定性显著优于复杂系统
- 仲裁机制对评分有系统性影响

### 不确定点
- Mock 评分逻辑与真实模型行为差异
- 50 条样本的代表性（需真实老年叙事样本验证）
- 情绪唤醒度在真实场景中的作用可能被低估

---

## Implications

### 对 v0.6 架构的影响

1. **建议移除组件**: C1 (情绪唤醒度), C2 (动态比例), C6 (投票加权)
2. **建议简化组件**: C4 (六维→三维), C7 (重新设计仲裁阈值)
3. **建议保留组件**: C3 (LLM 事件提取), C5 (多 Agent 评委)

### 性能预期

若采用 `llm_only` 配置:
- 评分: +5.7 分 (56.76 → 62.48)
- 稳定性: 3x 提升 (std 4.22 → 1.38)
- 延迟: 持平 (0.01ms → 0.00ms)
- 复杂度: 大幅降低 (7 组件→2 组件)
