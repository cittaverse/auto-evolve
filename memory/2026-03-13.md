## Lessons Learned 沉淀

### 5 条核心原则

1. **先扫描，再执行**（研究类任务）
2. **方向确认 > 执行速度**（方向性决策）
3. **验证环境，再选方案**（技术方案选择）
4. **版本备份，再修改**（文档修改）
5. **外部依赖，主动追踪**（V 执行任务）

### 决策树

```
任务开始
  ↓
是否需要外部证据？→ 是 → 先扫描（学术 + 专利 + 竞品）
  ↓
是否方向性决策？→ 是 → 先确认 V
  ↓
是否需要特定工具？→ 是 → 先验证环境
  ↓
执行
  ↓
是否修改现有文件？→ 是 → 先备份 v1
  ↓
完成 → 主动汇报 + 更新 Notion
```
