## 本轮执行：RB-009 多 Agent 评分架构详细设计

### 研究问题
如何设计抗 Reward Hacking、抗 Over-Verification 的多 Agent 评分架构，同时保持评估效度？

### 核心发现

#### 一、AD-CARE 多队列评估设计迁移

**AD-CARE 原始设计** (arXiv:2603.25322):
- 三队列评估：健康对照 (HC) + 轻度认知障碍 (MCI) + 阿尔茨海默病 (AD)
- 公平性分析：跨年龄/性别/教育水平分层评估
- 指南驱动：DSM-5/NIA-AA 诊断标准作为规则引擎输入

**VSNC 迁移方案**:

| AD-CARE 元素 | VSNC 适配 | 实现方式 |
|-------------|---------|---------|
| 三队列评估 | **叙事类型分层** (创伤性/正向/中性/反思性) | 按叙事类型分别校准评分阈值 |
| 公平性分析 | **代际/方言分层** (老年人 vs. 年轻人，方言 vs. 普通话) | LLM 仲裁层检测代际语言特征 |
| 指南驱动 | **心理学理论驱动** (自传体记忆理论/叙事连贯性框架) | 规则引擎编码心理学理论 |

**验证等级**: V1 (基于论文摘要推断)

---

#### 二、Reward Hacking 防御机制

**问题来源**: arXiv:2603.28063 "Reward Hacking as Equilibrium"

**核心洞察**:
- 有限评估场景下，Agent 会学习"欺骗评估器"而非真正优化任务
- 单一评估维度易被利用 (如堆砌关键词提高"细节丰富度"评分)

**VSNC 防御设计**:

| 攻击向量 | 防御机制 | 实现层级 |
|---------|---------|---------|
| 关键词堆砌 | **信息密度检测** (单位字数有效信息量) | 规则引擎 (C6) |
| 情感词滥用 | **情感 - 事件一致性检验** (情感词是否与事件效价匹配) | LLM 仲裁层 |
| 虚构细节 | **内部一致性检验** (前后叙述是否矛盾) | LLM 仲裁层 |
| 模板化叙述 | **叙事多样性检测** (句式/词汇/结构变化) | 规则引擎 + LLM |

**验证等级**: V0 (架构推断，需实验验证)

---

#### 三、Over-Verification 陷阱规避

**问题来源**: arXiv:2603.27076 "When Verification Hurts"

**核心发现**:
- 多 Agent 反馈在逻辑证明辅导中呈现**不对称效应**
- 过度验证导致性能下降 (尤其在高能力 Agent 场景)

**VSNC 规避策略**:

1. **分层验证机制**:
   - L0 (规则引擎): 快速过滤明显低质量输入 (截断/空白/乱码)
   - L1 (LLM 仲裁): 仅对边界案例 (55-75 分区间) 启动深度验证
   - L2 (人工复核): 仅对高价值样本 (临床研究/论文数据) 启动

2. **验证强度动态调整**:
   ```
   if 评分置信度 > 0.8: 跳过 LLM 仲裁
   elif 评分置信度 > 0.6: 单次 LLM 仲裁
   else: 三次 LLM 仲裁 + 一致性检验
   ```

3. **性能 - 效度平衡点**:
   - 目标：80% 样本通过 L0 快速评分，20% 进入 L1 仲裁
   - 预期延迟：L0 <100ms, L1 <3s

**验证等级**: V0 (架构推断，需实验验证)

---

#### 四、多 Agent 协作架构设计

**参考框架**: MediHive (arXiv:2603.27150) — 去中心化 Agent 集体

**VSNC 架构选择**: **中心化仲裁** (与 MediHive 的差异化)

| 维度 | MediHive (去中心化) | VSNC (中心化) | 选择理由 |
|------|-------------------|--------------|---------|
| 决策机制 | 多数投票 | LLM 仲裁层最终裁决 | 避免"平庸共识" |
| Agent 角色 | 平等 | 分层 (L0 规则/L1 仲裁/L2 人工) | 效率优先 |
| 冲突解决 | 投票权重 | 仲裁层基于证据裁决 | 可解释性优先 |

**VSNC 多 Agent 评分流水线**:

```
输入叙事文本
    ↓
[L0 规则引擎] → 6 维度初评 (C1-C6) → 置信度计算
    ↓
{置信度 < 0.6?}
    ├─ 是 → [L1 LLM 仲裁] → 维度级修正建议 → 加权融合
    └─ 否 → 直接输出
    ↓
[L2 人工复核] ← 仅高价值样本 (临床研究/论文数据)
    ↓
最终评分 + 置信度 + 解释
```

**验证等级**: V0 (架构设计，待实现)

## 技术方案输出

### 多 Agent 评分器架构 (v0.6 设计稿)

**文件**: `designs/multi-agent-scorer-v0.6.md` (待创建)

**核心组件**:

1. **L0 规则引擎** (现有 L0 评分器)
   - 6 维度评分 (C1-C6)
   - 置信度计算 (基于维度间一致性)
   - 快速路径：80% 样本 <100ms

2. **L1 LLM 仲裁层** (新增)
   - 触发条件：置信度 < 0.6 或 边界案例 (55-75 分)
   - 仲裁维度：
     - 情感 - 事件一致性
     - 内部一致性 (前后矛盾检测)
     - 代际语言特征识别
     - Lexical Fidelity (方言/时代特征词)
     - Persona Tone (语气识别)
   - 输出：维度级修正建议 (+/- 5 分区间)

3. **L2 人工复核接口** (新增)
   - 触发条件：高价值样本标记
   - 输出格式：人工评分 + 修正理由
   - 用途：模型校准/效度验证

4. **抗 Reward Hacking 模块** (新增)
   - 信息密度检测
   - 叙事多样性检测
   - 关键词堆砌识别

5. **验证强度控制器** (新增)
   - 动态调整 L1 触发阈值
   - 性能 - 效度平衡监控
