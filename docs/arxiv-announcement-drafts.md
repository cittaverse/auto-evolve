## 版本 3: 知乎（深度解析，中文社区向）

**标题**: 如何自动评估老年人回忆叙述的质量？我们开源了一个中文叙事评分系统

**正文**:

## 背景：为什么需要自动评分？

回忆疗法（Reminiscence Therapy）是让老年人讲述人生故事，从而改善认知功能和心理健康的干预方法。大量元分析证实，它对 MCI（轻度认知障碍）患者有中等程度的益处。

但问题在于：**如何评估"讲得好不好"？**

传统做法是人工编码——训练有素的研究员逐字阅读转录文本，按照标准化量表打分。问题：
- **贵**：专业编码员 200-500 RMB/小时
- **慢**：30 分钟录音需要 60-90 分钟编码
- **主观**：不同编码员之间一致性有限（Cohen's κ 通常 0.6-0.7）

这导致回忆疗法难以规模化——你不可能给 1000 个老人做人工编码。

## 我们的方案：神经符号 AI

过去 6 个月，我们构建了 CittaVerse Narrative Scorer v0.5。核心思路是：**用规则 + 启发式替代黑箱 LLM**。

### 六维度评估框架

我们从自传体记忆研究和叙事心理学文献中提炼了 6 个维度：

| 维度 | 测量什么 | 示例特征 |
|------|----------|----------|
| Event Richness | 叙述中包含多少独立事件 | 事件分段数、独特实体数 |
| Temporal Coherence | 时间线是否清晰 | 时间词密度、时间顺序一致性 |
| Causal Coherence | 事件间因果逻辑 | 因果连接词、解释性语句 |
| Emotional Depth | 情感表达深度 | 情感词多样性、自我表露程度 |
| Identity Integration | 与自我身份的关联 | 第一人称使用、价值观陈述 |
| Information Density | 信息分布是否均匀 | 冗余度、新信息比率 |

### 为什么不用 LLM？

LLM 当然可以评分，但有几个问题：
1. **成本**：每次调用$0.01-0.10，大规模使用不经济
2. **延迟**：1-5 秒响应，无法实时反馈
3. **黑箱**：无法解释"为什么给这个分数"
4. **离线不可用**：养老机构网络条件有限

我们的神经符号方案：
- **规则透明**：每条评分可追溯至具体特征
- **零 API 成本**：本地运行，只需 CPU
- **<100ms 延迟**：实时反馈
- **可定制**：研究者可根据需要调整权重

### 中文优化

现有叙事评估工具大多是英文的。中文有独特挑战：
- 时间表达更灵活（"前年大前年"vs."two years ago"）
- 因果连接词更隐晦（"就"、"才"的多义性）
- 情感表达更含蓄

我们构建了专用词表：
- 时间词：300+ 条目（包含方言变体）
- 因果连接词：80+ 条目
- 情感词：基于 HowNet 扩展至 2000+ 条目

## 临床验证

我们正在进行一项随机对照试点研究（RCT Pilot）：
- **样本量**：N=50（目标）
- **干预周期**：2 周，每日回忆练习
- **评估指标**：叙事评分变化 + MoCA + GDS-15
- **预期完成**：2026 Q2

初步反馈（未正式分析）：
- 参与者对即时评分反馈接受度高
- 评分与临床医生主观评估相关性约 0.75（n=12 初步样本）

## 开源与社区

代码采用 MIT License，公开于 GitHub：
- 核心评分引擎：Python 3.10+
- 中文词表：独立 JSON 文件，可扩展
- 示例数据：10 条匿名化叙事样本

我们希望这个工具能：
1. 帮助研究者快速验证回忆疗法效果
2. 降低数字疗法创业者的技术门槛
3. 推动中文老年心理健康评估的标准化
