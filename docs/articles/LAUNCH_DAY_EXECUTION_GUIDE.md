## 二、发布日详细流程 (D-0)

**检查清单**:

**常见错误**:
- ❌ 链接忘记替换 `[知乎/公众号链接]`
- ❌ GitHub 用户名写错
- ❌ 配图裂图 (需上传到知乎图床)
- ❌ 话题标签忘记添加

### 19:30 - 预览文章

**预览步骤**:
1. 知乎 App → 草稿箱 → 预览

   - 代码块格式是否正确
   - 表格是否对齐
   - 图片是否清晰

## 五、风险预案

### 情况 1: 发布后流量远低于预期
**症状**: D+1 阅读 < 500
**应对**:
1. 检查话题标签是否正确
2. 增加站外分享 (朋友圈/群)
3. 主动在相关问题下评论引流 (不要直接发链接，先有价值评论)
4. 联系知乎大 V 互动 (可选)

### 情况 2: 出现负面评论/质疑
**症状**: 质疑技术真实性/数据造假
**应对**:
1. **不要**情绪化回应
2. 理性提供证据 (GitHub 链接/代码/数据)
3. 如持续攻击 → 举报 + 拉黑
4. 如合理质疑 → 承认不足 + 改进计划

### 情况 3: GitHub 访问问题
**症状**: 国内用户无法访问 GitHub
**应对**:
1. 准备 Gitee 镜像 (可选)
2. 在评论区回复：「如访问 GitHub 有困难，可加我微信获取资料」
3. 准备 PDF 版本文章 (可选)

### 情况 4: 知乎审核不通过
**症状**: 文章被锁/需要修改
**应对**:
1. 检查是否包含敏感词 (政治/医疗/金融)
2. 检查外链是否过多 (知乎限制)
3. 修改后重新提交
4. 如持续不通过 → 转战公众号/掘金/思否
