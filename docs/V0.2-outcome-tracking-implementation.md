### 3.2 Layer 1：自动化数字标志物（核心创新）

**每次叙事会话自动提取，无需用户额外操作。**

| 标志物 | 提取方法 | 认知相关性 | 已有代码 |
|--------|---------|-----------|---------|
| **叙事连贯性** | 事件图连通度 | 执行功能 | ✅ v0.5 |
| **词汇丰富度** | TTR / 词频分析 | 语义记忆 | 可实现 |
| **时间定向** | 时间标记提取 | 定向力 | 可实现 |
| **情感词密度** | 情感词典匹配 | 情感处理 | ✅ v0.5 |
| **中心/外围比** | LLM 事件分类 | 注意力分配 | ✅ v0.5 |
| **信息密度** | 单位时间事件数 | 信息处理速度 | 可实现 |
| **自我指涉频率** | 第一人称代词 | 自我认同 | ✅ v0.5 |
| **重复率** | 跨会话主题重叠 | 工作记忆 | 需新增 |
| **语速变化** | ASR 时间戳 | 认知负荷 | 需 ASR |
| **停顿模式** | ASR 静音检测 | 词汇检索 | 需 ASR |

**关键洞察**：前 7 个标志物**已有代码基础**（叙事评分 v0.5），只需整合和纵向追踪。

### 3.3 Layer 2：标准化临床量表

| 量表 | 频率 | 实现方式 | 时长 |
|------|------|---------|------|
| **MoCA-B** (基础认知) | 基线 + 每 4 周 | 数字化问卷 | 10 分钟 |
| **GDS-15** (抑郁筛查) | 基线 + 每 2 周 | 数字化问卷 | 5 分钟 |
| **QOL-AD** (生活质量) | 基线 + 每 4 周 | 数字化问卷 | 5 分钟 |
| **SWLS** (生活满意度) | 基线 + 每 4 周 | 数字化问卷 | 3 分钟 |
| **NPS** (产品满意度) | 每 2 周 | 1 题 | 30 秒 |

**实现要点**：
- MoCA-B 需要授权（非免费），可用 Mini-Cog 或 GPCOG 替代（免费）
- GDS-15 公开可用，无需授权
- 量表数字化需符合原始信效度要求

class NarrativeSession:
    user_id: str
    session_id: str
    timestamp: datetime
    
    # Layer 1: 自动标志物
    narrative_score: float          # 综合叙事评分

    emotional_depth: float          # 情感深度

    information_density: float      # 信息密度

    mini_cog_score: Optional[int]

    # Layer 3: 行为数据
    session_duration_min: float

class LongitudinalReport:
    user_id: str

### 6.2 纵向变化检测算法

```python
def detect_cognitive_change(sessions: List[NarrativeSession], 
                            window_weeks: int = 4) -> Dict:
    """检测认知变化趋势"""
    
    # 1. 计算基线（前 2 周平均）
    baseline = calculate_baseline(sessions[:4])
    
    # 2. 计算当前（最近 2 周平均）
    current = calculate_current(sessions[-4:])
    
    # 3. 计算变化
    changes = {}
    for marker in COGNITIVE_MARKERS:
        delta = current[marker] - baseline[marker]
        # 标准化为效应量 (Cohen's d)
        effect_size = delta / baseline_std[marker]
        changes[marker] = {
            'delta': delta,
            'effect_size': effect_size,
            'direction': 'improved' if delta > 0 else 'declined',
            'significance': 'significant' if abs(effect_size) > 0.5 else 'marginal'
        }
    
    # 4. 综合评估
    improved = sum(1 for c in changes.values() if c['direction'] == 'improved')
    declined = sum(1 for c in changes.values() if c['direction'] == 'declined')
    
    return {
        'markers': changes,
        'overall_trend': 'improving' if improved > declined else 'stable' if improved == declined else 'declining',
        'confidence': calculate_confidence(sessions)
    }
```

## 七、竞争壁垒分析

### 7.1 为什么这是壁垒

| 壁垒维度 | 说明 |
|---------|------|
| **数据壁垒** | 叙事+认知标志物纵向数据，竞品从零开始需要 6-12 个月 |
| **算法壁垒** | 叙事评分 v0.5（6 维度 + 中心/外围 + 情绪唤醒）是原创方法 |
| **临床证据壁垒** | Pilot RCT 数据一旦产出，是最硬的护城河 |
| **专利壁垒** | "叙事质量自动评分 → 认知标志物"这条链路无现有专利覆盖 |

### 7.2 竞品追赶难度

| 竞品 | 追赶难度 | 原因 |
|------|---------|------|
| Recall Aid | 高 | 无评分系统，无纵向数据 |
| 传统 RT 平台 | 极高 | 无 AI 能力 |
| VR 回忆疗法 | 中 | 有技术能力但方向不同 |
| 大厂（腾讯/字节） | 中 | 有技术但缺临床理解 |

## 十、证伪条件

什么证据会推翻本方案？

1. **叙事标志物与认知量表相关性 r < 0.3** → 标志物无效，需换方法
2. **用户量表完成率 < 30%** → 量表方案不可行，需纯被动采集
3. **Pilot 8 周无显著变化** → 干预周期不够，或效果不存在
