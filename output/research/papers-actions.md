## 二、详细行动计划

### 🔴 P0-1: 双重记忆架构原型 (2 周)

**目标**: 实现 Working Memory + Experience Memory 双通道

**具体任务**:
```
Week 1:
├── 设计 Working Memory 数据结构
│   ├── 当前对话状态
│   ├── 短期回忆线索 (最近 3 轮)
│   └── 用户情绪状态
├── 设计 Experience Memory 数据结构
│   ├── 集体记忆锚点 (年代/地域/事件)
│   ├── 历史回忆模式 (用户偏好)
│   └── 叙事质量评分历史
└── 实现双记忆检索接口

Week 2:
├── 集成到阿宝对话流程
├── A/B 测试设计
└── 效果评估指标定义
```

**成功指标**:
- 回忆相关度提升 30%+
- 对话连贯性评分提升 20%+

**负责人**: Core (工程实现) + Hulk (效果评估)

### 🔴 P0-2: 记忆复用机制 (1 周)

**目标**: 降低 LLM 调用成本，复用历史回忆

**具体任务**:
```
Day 1-2: 回忆片段向量化存储
├── 设计回忆片段切分策略
├── 选择嵌入模型 (bge-m3 或 text-embedding-3)
└── 构建向量索引 (Qdrant/Milvus)

Day 3-4: 检索与复用逻辑
├── 相似回忆检索 (top-k)
├── 复用决策阈值 (相似度>0.85 直接复用)
└── 部分复用策略 (片段级复用)

Day 5-7: 集成与测试
├── 集成到对话流程
├── 成本监控埋点
└── 质量评估 (复用 vs 新生成)
```

**成功指标**:
- LLM 调用次数减少 50%+
- 用户无明显感知质量下降

**负责人**: Core

### 🟠 P1-1: 中文回忆评估基准 (4 周)

**具体任务**:
```
Week 1: 数据收集设计

Week 4: 基准构建与论文准备
├── 构建评估指标体系

```

**成功指标**:
- 数据集规模 1000+ 会话
- 论文投稿完成
- 技术差异化优势建立

**负责人**: Hulk (研究) + V (用户招募)

### 🟠 P1-2: MCTS 回忆挖掘 (2 周)

**目标**: 实现动态回忆探索路径，替代固定追问模板

**具体任务**:
```
Week 1: MCTS 框架设计
├── 定义状态空间 (情境/感官/情感/意义)
├── 定义动作空间 (追问/切换/回溯/结束)
├── 设计奖励函数 (回忆丰富度 + 用户满意度)
└── 实现 MCTS 搜索算法

Week 2: 集成与调优
├── 集成到回忆挖掘模块
├── 超参数调优 (探索系数/模拟次数)
└── 与固定模板 A/B 测试
```

**成功指标**:
- 回忆深度提升 (平均对话轮次 +30%)
- 用户满意度提升 (NPS +10)

**负责人**: Core
