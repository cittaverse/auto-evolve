### 行动 1: 事件驱动用户画像更新

**灵感来源**: PERMA (2603.23231)  
**验证状态**: V1 (单来源确认)  
**优先级**: P0  
**工时**: 2 周

**当前问题**:
- 阿宝用户画像可能过于静态（基于初期问答）
- 未充分利用日常对话中的事件信息
- 偏好更新依赖显式询问，而非隐性推断

**实施方案**:
```
Phase 1 (Week 1): 事件提取器
  ├─ 从对话轮次提取事件元数据
  │   ├─ 时间线索 (年代、季节、人生阶段)
  │   ├─ 地点线索 (家乡、工作地、迁移)
  │   ├─ 人物线索 (家人、朋友、同事)
  │   ├─ 情感价 (正向/负向/中性)
  │   └─ 事件类型 (成就、挫折、转折、日常)
  └─ 存储为事件图谱 (而非扁平标签)

Phase 2 (Week 2): 偏好推断引擎
  ├─ 高频事件类型 → 核心价值推断
  ├─ 高情感强度事件 → 身份锚点标记
  ├─ 跨 session 一致性检查
  └─ 渐进式画像更新 (加权融合，非覆盖)

Phase 3 (Week 3): 验证与迭代
  ├─ A/B 测试：事件驱动 vs 静态画像
  ├─ 指标：回忆引导相关性、用户满意度
  └─ 人工审核：推断准确性抽样检查
```

### 行动 2: 回忆证据分级系统

**灵感来源**: ElephantBroker (2603.25097)  
**验证状态**: V3 (架构复核，2200+ 测试通过)  
**优先级**: P0  
**工时**: 3 周

**当前问题**:
- 阿宝存储的回忆内容无可信度区分
- 用户/家属无法识别哪些是"确凿回忆"vs"模型推断"
- 研究导出数据缺乏证据等级标注

**实施方案**:
```
Phase 1 (Week 1): 证据来源分类器
  ├─ S1: 照片/实物佐证 (最高可信)
  ├─ S2: 多次一致提及 (跨 3+ 对话)
  ├─ S3: 单次自述 (标准)
  └─ S4: 模型推断填充 (最低，需标记)

Phase 2 (Week 2): 存储与展示
  ├─ 内部存储：每条回忆带 evidence_level 字段
  ├─ 用户展示：人生时间线用颜色区分证据等级
  └─ 导出功能：研究数据包含证据等级

Phase 3 (Week 3): 温和验证机制
  ├─ S4 内容触发温和确认："这个细节我理解得对吗？"
  ├─ S2→S1 升级：邀请用户上传照片佐证
  └─ 矛盾检测：同一事件不同陈述标记不一致
```

**技术细节**:
```python
# 证据等级判定逻辑
def classify_evidence(recall_text, context):
    if context.has_photo_attachment:
        return "S1"
    elif context.mention_count >= 3 and context.consistency > 0.9:
        return "S2"
    elif context.mention_count == 1:
        return "S3"
    elif recall_text.is_model_inferred:
        return "S4"
    else:
        return "S3"

# 用户界面展示
evidence_colors = {
    "S1": "#22c55e",  # 绿色 - 确凿
    "S2": "#3b82f6",  # 蓝色 - 可靠
    "S3": "#f59e0b",  # 黄色 - 自述
    "S4": "#ef4444"   # 红色 - 待确认
}
```

**预期收益**:
- 提升透明度，增强用户信任
- 研究数据质量提升（可筛选高证据等级样本）
- 早期认知障碍信号检测（S4 比例异常升高）

**负责人**: @Core  
**验收标准**:
- 证据分类准确率 > 85%
- 用户对透明度满意度 > 4.5/5
- 研究导出数据 100% 包含证据等级

### 行动 3: 碎片回忆整合优化

**灵感来源**: AD-CARE (2603.25322)  
**验证状态**: V1 (单来源确认，3 队列 N=2000+ 验证)  
**优先级**: P0  
**工时**: 1 周

**当前问题**:
- 老人回忆天然是碎片化的
- 当前阿宝可能过度追问完整细节
- 缺失信息未显式标记，导致后续混淆

**实施方案**:
```
Phase 1 (Week 1): 不完整输入检测
  ├─ 识别回忆陈述中的缺失槽位
  │   ├─ 时间模糊 ("那时候"→具体年份？)
  │   ├─ 人物模糊 ("他"→具体是谁？)
  │   ├─ 地点模糊 ("那里"→具体哪里？)
  │   └─ 事件模糊 ("那件事"→具体发生了什么？)
  └─ 显式标记不确定部分 (内部存储)

Phase 2 (Week 1): 渐进丰富策略
  ├─ 不强制单次补全
  ├─ 跨多轮对话自然补充
  ├─ 与已有锚点关联 (而非孤立存储)
  └─ 阿宝话术优化

Phase 3 (Week 2): 可视化与导出
  ├─ 人生时间线显示"待补充"标记
  └─ 支持用户主动补充历史回忆
```

**阿宝话术示例**:
```
❌ 过度追问: "您刚才说 1985 年的事，具体是哪个月？
            当时还有谁在场？具体说了什么？"

✅ 渐进丰富: "您刚才提到 1985 年那次经历，
            听起来对您很重要。如果以后您想
            起更多细节，随时可以告诉我。
            今天我们已经聊了很多了。"

✅ 锚点关联: "您提到的这件事，让我想起您之前
            说过的 [相关事件]。这两件事之间
            有什么联系吗？"
```

**预期收益**:
- 用户体验显著提升（减少追问压力）
- 回忆质量提升（自然补充 vs 被迫回忆）
- 符合老年认知特点（碎片化是正常的）

**负责人**: @Core  
**验收标准**:
- 用户"被打断感"评分降低 40%+
- 单次对话时长减少 20%（效率提升）
- 跨 session 回忆补充率 > 30%
