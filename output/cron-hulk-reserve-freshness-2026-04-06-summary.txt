### 1. arXiv:2603.10062v2 — Multi-Agent Memory from a Computer Architecture Perspective

**提交日期**: 2026-03-09 (v1), **修订日期**: 2026-03-30 (v2) ← **7 天内更新**  
**来源**: arXiv cs.AR/cs.AI/cs.MA  
**验证等级**: V1 (单来源确认)

**核心论点**:
- 将多 Agent 记忆框架化为计算机架构问题
- 提出三层记忆层次：I/O、cache、memory
- 识别两个关键协议缺口：跨 Agent 缓存共享、结构化记忆访问控制
- **最紧迫的开放挑战**: 多 Agent 记忆一致性 (memory consistency)

**与 Hulk 现有结论的对比**:

| Hulk 现有结论 (GEO #100, 2026-04-03) | 新论文观点 | 一致性评估 |
|-------------------------------------|-----------|-----------|
| 四层架构：Working/Episodic/Semantic/Procedural | 三层架构：I/O/Cache/Memory | 🟡 **部分一致** — 分类维度不同 (功能 vs. 架构) |
| Working Memory 用于 session 级缓存 | Cache layer 用于跨 Agent 共享 | 🟢 **一致** — 缓存概念相同，新论文强调跨 Agent 场景 |
| Episodic + Semantic 分离存储 | Memory layer 统一存储 | 🟡 **部分一致** — Hulk 设计更细粒度 |
| Procedural Memory 封装技能调用 | 未明确涉及 | ⚪ **无冲突** — 新论文未覆盖技能层 |
| 未明确讨论 memory consistency | **核心挑战**: memory consistency | 🔴 **缺口** — Hulk 设计未考虑一致性问题 |

**对 Hulk 设计的影响**:
1. **需补充**: 在四层架构设计中加入 memory consistency 机制 (如版本控制、冲突检测)
2. **需澄清**: Working Memory 的 session 隔离 vs. 跨 Agent 缓存共享的边界
3. **可借鉴**: 三层架构的 I/O 层概念 → 可映射到 Hulk 的 Episodic Memory 输入/输出接口

**建议行动**: 更新 `designs/agent-memory-four-layer-architecture.md`，增加 memory consistency 章节。
