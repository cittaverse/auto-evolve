## 一、EM-LLM：人类启发式情景记忆架构

### 技术概述

**来源**: Fountas et al., ICLR 2025, arXiv:2407.09450v3  
**机构**: 华为 Noah's Ark Lab + UCL

**核心创新**:
```
┌─────────────────────────────────────────────────────────┐
│                    EM-LLM 架构                          │
├─────────────────────────────────────────────────────────┤
│  1. 在线事件分段                                         │
│     - Bayesian Surprise 检测语义突变                      │
│     - 图论边界优化 (动态规划)                             │
│     - 输出：连贯的"情景事件"序列                          │
│                                                         │
│  2. 两阶段记忆检索                                       │
│     - 阶段 1: 语义相似度检索 (top-k candidates)          │
│     - 阶段 2: 时间相邻性检索 (temporal contiguity)       │
│     - 模拟人类记忆的"时间邻近效应"                        │
│                                                         │
│  3. 无需微调集成                                         │
│     - 即插即用，无需 fine-tuning                         │
│     - 兼容主流开源 LLM (Llama, Qwen, GLM 等)              │
└─────────────────────────────────────────────────────────┘
```

### 性能表现

| 基准测试 | EM-LLM | SOTA (InfLLM) | 提升 |
|---------|--------|---------------|------|
| LongBench | 68.4 | 61.2 | +11.8% |
| ∞-Bench (10M tokens) | 54.7 | 38.9 | +40.6% |
| 事件边界检测 (vs 人类) | κ=0.78 | - | 高度一致 |

### VSNC/L0 集成路径

**适用场景**:
- 用户人生故事的自动分段 (按事件/主题)
- 跨会话长期记忆检索 (回忆触发)
- 叙事连贯性评估 (事件边界流畅度)

**集成步骤**:
```
Phase 1 (1 月): PoC 验证
├─ 部署 EM-LLM 开源实现
├─ 使用现有用户叙事数据测试分段效果
└─ 对比 v0.6 规则分段 vs EM-LLM 分段

Phase 2 (2-3 月): 产品集成
├─ 集成到 abao-recall-miner skill
├─ 新增"事件边界可视化"功能
└─ A/B 测试分段质量对用户满意度的影响

Phase 3 (4-6 月): 专利布局
├─ 申请"事件分段 + 检索"方法专利
├─ 论文投稿 (ACL/EMNLP 2026)
└─ 开源贡献 (Hugging Face integration)
```

**验证等级**: V2 (项目页面 + arXiv + OpenReview 交叉确认)

## 二、神经符号 AI：可审计临床评估框架

### 技术概述

**来源**: Nature Communications Medicine, 2025  
**标题**: "Neuro-symbolic AI for auditable cognitive information extraction from clinical narratives"

**核心论点**:
> 独立 LLM 存在黑箱、不可审计、幻觉风险。神经符号 AI 提供安全自动化临床数据提取路径，在结构化医学本体 (SNOMED CT, ICD-11) 内运行。

### CittaVerse 架构对齐

| 维度 | NeSy 最佳实践 | CittaVerse L0 | 对齐度 |
|------|--------------|--------------|--------|
| **方法** | 神经网络 + 符号规则 | 规则评分 + LLM 仲裁 | ✅ 完全一致 |
| **可解释性** | 规则可追溯 | 每维度评分可解释 | ✅ 完全一致 |
| **幻觉控制** | 结构化本体约束 | 规则层硬约束 | ✅ 完全一致 |
| **临床适用** | 可审计决策 | 临床评估工具 | ✅ 完全一致 |

### 差异化优势

基于文献分析，CittaVerse 是**唯一**同时满足以下条件的自传体记忆评估工具：

```
┌─────────────────────────────────────────────────────────┐
│              CittaVerse 差异化定位                       │
├─────────────────────────────────────────────────────────┤
│  ✅ 中文优化 (vs 学术界/竞品英文为主)                     │
│  ✅ 神经符号架构 (vs 纯 LLM 黑箱)                         │
│  ✅ 临床评估场景 (vs 内容生产工具)                        │
│  ✅ 可离线部署 (vs 纯云端 API)                           │
│  ✅ 开源免费 (vs 高订阅成本)                             │
└─────────────────────────────────────────────────────────┘
```

**验证等级**: V1 (Nature Comm Med 单来源确认)

## 三、持久化用户画像记忆框架

### 技术概述

**来源**: Westhäußer et al., arXiv:2510.07925 (2025-10)  
**标题**: "Towards Persistent Memory and User Profiling for Personalized LLM Agents"

**核心创新**:
- **统一个性化定义**: 将个性化作为概念基础，推导自适应 LLM Agent 技术需求
- **持久化记忆**: 整合多源检索、动态协调、自验证机制
- **演进式用户画像**: 支持长期交互中的用户偏好/状态演化
- **五日用戶研究**: 初步验证用户对感知个性化的反馈

### VSNC/L0 集成路径

**当前状态**: abao-profile 系统已收集 preferred_name, birth_year, hometown, family_role

**升级方向**:
```
v0.7 画像系统升级
├─ 动态偏好追踪 (对话中自动更新)
├─ 记忆 - 画像联动 (回忆内容反哺画像)
├─ 跨会话一致性 (长期交互中的身份连续)
└─ 用户可控性 (用户可审查/修改画像)
```

**评估指标**:
- 检索准确率 (vs 用户真实偏好)
- 回应正确性 (用户满意度评分)
- BertScore (个性化回应质量)

**验证等级**: V1 (arXiv 预印本)

## 四、Eye2Recall：多模态回忆交互设计

**核心创新**:

- **专家访谈 + 用户研究**: 10 位老年参与者评估

## 七、LLM-as-Judge 评估框架

### 方法论综述

**来源**: arXiv:2412.05579 (2024-12)  
**标题**: "LLMs-as-Judges: A Comprehensive Survey on LLM-based Evaluation Methods"

**三种核心方法**:
| 方法 | 描述 | CittaVerse 应用 |
|------|------|---------------|
| Pointwise scoring | 单一输出独立评分 | ✅ L0 仲裁机制 |
| Pairwise comparison | 多输出对比排序 | 未来多版本对比 |
| Pass/fail checks | 二元决策 | 合规性检查 |

### CittaVerse v0.6 仲裁机制设计

```
LLM 仲裁流程
├─ 输入：规则层各维度评分 + 叙事文本
├─ Prompt 设计：
│   ├─ 清晰评估标准 (rubric-based)
│   ├─ 结构化输出 (JSON)
│   └─ 偏差缓解指令 (位置/长度偏差)
├─ 仲裁逻辑：
│   ├─ 规则评分冲突 → LLM 裁决
│   ├─ 边界案例 → LLM 加权
│   └─ 置信度输出 → 低置信度标记人工复核
└─ 输出：最终评分 + 置信度 + 解释
```

**验证等级**: V1 (arXiv 综述)
