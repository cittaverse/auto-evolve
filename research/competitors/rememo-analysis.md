## 2. 核心技术实现

### 2.1 系统架构

```
┌─────────────────────────────────────────────────────────┐
│                    Rememo 系统架构                       │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌─────────────┐    ┌──────────────┐    ┌────────────┐ │
│  │   照片输入   │ →  │  生成式 AI   │ →  │  问题建议  │ │
│  │  (老照片)   │    │  (多模态)    │    │  (治疗师)  │ │
│  └─────────────┘    └──────────────┘    └────────────┘ │
│                          ↓                               │
│                   ┌──────────────┐                       │
│                   │  人类治疗师   │                       │
│                   │  (关系性工作) │                       │
│                   └──────────────┘                       │
│                          ↓                               │
│                   ┌──────────────┐                       │
│                   │   会话记录    │                       │
│                   │  (AI 整理)    │                       │
│                   └──────────────┘                       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 2.2 关键技术组件

| 组件 | 技术 | 说明 |
|------|------|------|
| **图像理解** | 多模态 VLM | 识别照片内容、人物、场景、时代线索 |
| **问题生成** | 生成式 LLM | 基于照片内容生成回忆引导问题 |
| **语境适配** | 本地化微调 | 适应新加坡文化背景 (多语言、多元文化) |
| **治疗师界面** | Web/Mobile | 展示 AI 建议，支持治疗师修改/选择 |
| **会话记录** | ASR + NLP | 录音转文字，自动整理关键内容 |

### 2.3 核心设计理念

1. **AI-in-the-loop** (非 AI-first)
   - AI 是支持工具，不是主导者
   - 治疗师保留最终决策权
   - AI 建议可被修改、拒绝、忽略

2. **关系性工作优先**
   - 承认治疗师与患者的关系是疗效核心
   - AI 不尝试取代人类情感连接
   - AI 处理机械性工作 (内容生成、记录整理)

3. **语境敏感**
   - 尊重新加坡多元文化背景
   - 适应当地护理基础设施限制
   - 考虑语言多样性 (英语/华语/马来语/泰米尔语)

## 3. 与 CittaVerse 对比

### 3.1 定位对比

| 维度 | Rememo | CittaVerse (阿宝) |
|------|--------|------------------|
| **核心目标** | 痴呆症回忆疗法 | 回忆疗法 + 人生故事书 |
| **产品定位** | 治疗师工具 (B 端) | AI Copilot + 人类引导 (B2B2C) |
| **用户角色** | 治疗师主导 | 引导员/家属主导 |
| **输入形式** | 照片 → 问题 | 照片 → 引导回忆 |
| **输出形式** | 会话记录 | 人生故事书 + 六维评分 |
| **核心理念** | AI 支持人类引导 | 增强人类引导，非替代 |
| **地理语境** | 新加坡 | 中国 (方言、集体记忆) |
| **商业化** | 研究阶段 | 产品化中 (V0.2) |

### 3.2 技术对比

| 技术维度 | Rememo | CittaVerse | 优劣分析 |
|----------|--------|------------|----------|
| **多模态输入** | ✅ 照片理解 | ✅ 照片理解 | 相当 |
| **问题生成** | ✅ 生成式 AI | ✅ 生成式 AI | 相当 |
| **方言支持** | ⚠️ 多语言 (新加坡) | ✅ 中文方言优化 | CittaVerse 更专注 |
| **叙事评分** | ❌ 无 | ✅ L0 六维认知特征 | CittaVerse 领先 |
| **集体记忆** | ⚠️ 新加坡本地化 | ✅ 中国集体记忆锚点 | CittaVerse 更深厚 |
| **临床验证** | ⚠️ 研究阶段 | 🟡 Pilot 准备中 | 均需验证 |
| **输出产物** | 会话记录 | 人生故事书 | CittaVerse 更完整 |

### 3.3 差异化机会

| 机会 | 具体策略 | 优先级 |
|------|----------|--------|
| **本土化深度** | 中国方言 (粤语/吴语/闽南语等) + 集体记忆锚点 (改革开放/文革/知青等) | P0 |
| **双轨模式** | B 端 (机构) + C 端 (家庭) vs. Rememo 仅 B 端 | P0 |
| **叙事评分** | L0 六维认知特征作为客观评估工具 | P0 |
| **人生故事书** | 可交付的完整产物 vs. 会话记录 | P1 |
| **开源策略** | 部分开源建立学术影响力 | P1 |
