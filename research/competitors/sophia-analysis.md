### 2.1 System 3 理论框架

### 2.2 System 3 核心组件

| 组件 | 功能 | 技术实现 |
|------|------|----------|
| **Narrative Identity** | 维持身份连续性 | 叙事记忆存储 + 检索 |
| **Narrative Memory** | 自传体记忆组织 | 非 RAG，叙事性结构 |
| **User/Self Modeling** | 用户与自我建模 | 持续更新的表征 |
| **Hybrid Reward System** | 混合奖励系统 | 内在 + 外在奖励 |

### 2.3 四大协同机制

1. **Process-Supervised Thought Search**
   - 过程监督的搜索策略
   - 监控推理过程质量

2. **Narrative Memory (叙事记忆)**
   - **核心创新**: 非简单 RAG，而是叙事性组织
   - 将重复性推理转化为自驱动的自传体过程
   - 维持身份连续性

3. **User and Self Modeling**
   - 持续更新的用户模型
   - 自我模型 (agent 自身能力/局限认知)

4. **Hybrid Reward System**
   - 内在奖励 (好奇心、一致性)
   - 外在奖励 (任务成功、用户反馈)

## 8. 对 CittaVerse 的启示

### 8.1 架构设计启示

1. **叙事记忆 ≠ RAG**
   - 当前阿宝记忆可能是简单 RAG 检索
   - Sophia 启示：应按叙事结构组织 (章节、主题、人物、情感)
   - 这样更能维持用户身份连续性

2. **System 3 元认知层**
   - 阿宝可增加自评估机制
   - 定期反思对话质量、用户状态、干预效果
   - 自驱动改进对话策略

3. **混合奖励系统**
   - 当前 L0 六维评分可作为外在奖励
   - 可增加内在奖励 (对话连贯性、用户参与度)
   - 用于优化对话引导策略

### 8.2 产品设计启示

1. **人生故事书作为记忆载体**
   - 不仅是产品输出，也是阿宝记忆的核心组件
   - 故事书结构 = 叙事记忆结构
   - 支持持续更新和检索

2. **身份连续性维持**
   - 阿宝应记住用户的核心身份叙事
   - 在对话中引用和强化这些叙事
   - 帮助 MCI 患者维持自我认同感
