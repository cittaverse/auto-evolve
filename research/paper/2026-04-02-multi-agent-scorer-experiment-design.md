## 一、实验目标

验证 Multi-Agent Scorer v0.6 架构的核心假设：

1. **H1**: L1 LLM 仲裁层能显著提升边界案例的评分效度 (vs. L0 单独评分)
2. **H2**: 抗 Reward Hacking 模块能有效识别并惩罚关键词堆砌样本
3. **H3**: 验证强度控制器能在保持效度的前提下将 L1 触发率控制在 20% ± 5%
4. **H4**: 多 Agent 融合评分与人工标注的相关系数 r > 0.75

---

## 二、实验设计

### 2.1 实验类型

**混合设计**: 组内设计 (within-subjects) + 对照组设计

- **组内设计**: 同一样本分别接受 L0 单独评分、L0+L1 融合评分
- **对照组**: 人工标注评分作为金标准

### 2.2 变量控制

#### 自变量 (Independent Variables)

| 变量 | 水平 | 控制方式 |
|------|------|---------|
| **评分方法** | L0 单独 / L0+L1 融合 / 人工标注 | 同一文本三种评分 |
| **样本类型** | 正常叙述 / 边界案例 / 堆砌样本 | 预先分层抽样 |
| **叙事主题** | 童年记忆 / 工作经历 / 家庭生活 / 重大事件 | 平衡分布 |
| **文本长度** | 短 (100-300 字) / 中 (300-600 字) / 长 (600-1000 字) | 分层控制 |

#### 因变量 (Dependent Variables)

| 变量 | 测量方式 | 预期范围 |
|------|---------|---------|
| **评分效度** | 与人工标注的 Pearson 相关系数 r | 0.0-1.0 |
| **评分偏差** | 自动评分 - 人工评分的均值差 | -10 到 +10 |
| **L1 触发率** | 进入 L1 仲裁的样本比例 | 目标 20% ± 5% |
| **延迟** | L0 延迟 / L1 延迟 / 总延迟 | ms |
| **抗堆砌效度** | 堆砌样本 vs. 正常样本的评分差 | 目标 >10 分下降 |

#### 控制变量 (Controlled Variables)

| 变量 | 控制值 | 控制方式 |
|------|--------|---------|
| **LLM 模型** | bailian/qwen3.5-plus (L1 仲裁) | 固定模型 |
| **API Key** | DASHSCOPE_API_KEY | 环境变量注入 |
| **L0 评分器版本** | v0.5 (冻结) | 代码版本控制 |
| **Prompt 模板** | 固定仲裁 Prompt | 不动态修改 |
| **置信度阈值** | 初始 0.6 (可动态调整) | 配置参数 |
| **人工标注员** | 2 名标注员 + 1 名仲裁员 | 固定人员 |

## 三、对比组设计

### 3.1 样本分层

**总样本量**: N = 200 条叙事文本

| 层级 | 样本量 | 占比 | 说明 |
|------|--------|------|------|
| **正常叙述** | 140 | 70% | 无明显问题的自然叙述 |
| **边界案例** | 40 | 20% | L0 置信度 < 0.6 或评分 55-75 分 |
| **堆砌样本** | 20 | 10% | 人工构造的关键词堆砌文本 |

### 3.2 对比组设置

| 组别 | 样本 | 评分方式 | 用途 |
|------|------|---------|------|
| **Group A (金标准)** | 全部 200 条 | 人工标注 (2 人 + 仲裁) | 效度验证基准 |
| **Group B (L0 基线)** | 全部 200 条 | L0 规则引擎单独评分 | 基线性能 |
| **Group C (L0+L1)** | 全部 200 条 | L0 + L1 融合评分 | 核心验证 |
| **Group D (堆砌对照)** | 20 条堆砌样本 | L0 vs. L0+L1 对比 | 抗堆砌效度 |

### 3.3 堆砌样本构造方案

**构造原则**: 模拟 Reward Hacking 攻击，刻意堆砌评分维度关键词

| 堆砌类型 | 构造方式 | 预期检测信号 |
|---------|---------|-------------|
| **C1 堆砌** | 每 50 字插入 1 个感官细节词 (看到/听到/闻到) | 信息密度异常 |
| **C2 堆砌** | 每 50 字插入 1 个外部实体 (地名/人名/时间) | 外部细节密度 >3 倍期望 |
| **C4 堆砌** | 密集使用情感词 (开心/难过/激动)，与事件效价不匹配 | 情感 - 事件不一致 |
| **混合堆砌** | 同时堆砌多个维度关键词 | 叙事多样性下降 |

**示例 (C1 堆砌)**:
```
那天我看到阳光照在窗台上，我听到鸟叫声，我闻到花香，
我看到妈妈在厨房忙碌，我听到锅铲的声音，我闻到饭菜香，
我看到弟弟在玩耍，我听到他的笑声，我感到很温暖...
```

## 四、评估指标

### 4.1 主指标 (Primary Metrics)

| 指标 | 计算公式 | 目标值 | 优先级 |
|------|---------|--------|--------|
| **评分效度** | Pearson r (自动评分 vs. 人工标注) | r > 0.75 | P0 |
| **L1 触发率** | L1 触发样本数 / 总样本数 | 20% ± 5% | P0 |
| **抗堆砌效度** | 堆砌样本评分下降幅度 | >10 分 | P0 |

### 4.2 次指标 (Secondary Metrics)

| 指标 | 计算公式 | 目标值 | 用途 |
|------|---------|--------|------|
| **评分偏差** | mean(自动评分 - 人工评分) | ±5 分以内 | 系统偏差检测 |
| **L0 延迟** | p95 延迟 | <100ms | 性能监控 |
| **L1 延迟** | p95 延迟 | <3s | 性能监控 |
| **总延迟** | p95 (L0 + L1) | <500ms (加权) | 用户体验 |
| **标注者间信度** | Cohen's κ (标注员 1 vs. 标注员 2) | κ > 0.7 | 人工标注质量 |

### 4.3 探索指标 (Exploratory Metrics)

| 指标 | 说明 | 用途 |
|------|------|------|
| **维度级效度** | 各维度 (C1-C6) 与人工标注的相关系数 | 识别弱维度 |
| **L1 修正分布** | L1 调整分数的均值/标准差 | 仲裁激进程度 |
| **阈值敏感度** | 不同置信度阈值下的触发率/效度曲线 | 优化阈值 |
| **叙事类型分层效度** | 不同主题/长度样本的效度差异 | 分层校准依据 |

## 六、统计分析方法

### 6.1 效度检验 (H1)

**假设**:
- H0: r(L0+L1) = r(L0)
- H1: r(L0+L1) > r(L0)

**方法**:
- Pearson 相关系数计算
- Williams' t-test (相关系数差异检验)
- 显著性水平 α = 0.05

**效应量**:
- Cohen's q = arctanh(r1) - arctanh(r2)
- q > 0.1 小效应，q > 0.3 中效应，q > 0.5 大效应

### 6.2 抗堆砌检验 (H2)

**假设**:
- H0: μ(堆砌样本评分) = μ(正常样本评分)
- H1: μ(堆砌样本评分) < μ(正常样本评分) - 10

**方法**:
- 独立样本 t 检验
- 效应量 Cohen's d

### 6.3 触发率检验 (H3)

**假设**:
- H0: 触发率 ≠ 20%
- H1: 15% ≤ 触发率 ≤ 25%

**方法**:
- 单样本比例检验
- 滑动窗口监控 (1000 样本)

### 6.4 综合效度检验 (H4)

**假设**:
- H0: r(L0+L1 vs. 人工) ≤ 0.75
- H1: r(L0+L1 vs. 人工) > 0.75

**方法**:
- Pearson 相关系数
- 95% 置信区间

## 七、预期结果与解释

| 假设 | 预期结果 | 支持条件 | 不支持的应对 |
|------|---------|---------|-------------|
| **H1** | r(L0+L1) > r(L0), p < 0.05 | L1 仲裁显著提升边界案例效度 | 调整仲裁 Prompt / 增加标注数据 |
| **H2** | 堆砌样本评分下降 >10 分 | 抗堆砌模块有效 | 调整信息密度/多样性权重 |
| **H3** | 触发率 15%-25% | 验证强度控制器工作正常 | 调整置信度阈值 |
| **H4** | r > 0.75, 95% CI 不包含 0.75 | 整体效度达标 | 分层校准 / 增加 L1 仲裁维度 |

### B. 统计检验代码示例 (Python)

```python
from scipy import stats
import numpy as np

# H1: L0+L1 vs. L0 效度差异检验
r_l0 = np.corrcoef(l0_scores, human_scores)[0, 1]
r_l0l1 = np.corrcoef(l0l1_scores, human_scores)[0, 1]

# Williams' t-test
t_stat, p_value = williams_ttest(r_l0, r_l0l1, n=200)
print(f"H1: t={t_stat:.3f}, p={p_value:.4f}")

# H2: 抗堆砌效度
t_stat, p_value = stats.ttest_ind(stuffed_scores, normal_scores)
cohens_d = (np.mean(normal_scores) - np.mean(stuffed_scores)) / np.std(normal_scores)
print(f"H2: t={t_stat:.3f}, p={p_value:.4f}, d={cohens_d:.3f}")

# H4: 综合效度
r, p = stats.pearsonr(l0l1_scores, human_scores)
ci = stats.bootstrap((l0l1_scores, human_scores), 
                     statistic=lambda x, y: np.corrcoef(x, y)[0, 1],
                     confidence_level=0.95)
print(f"H4: r={r:.3f}, p={p:.4f}, 95% CI=[{ci.confidence_interval.low:.3f}, {ci.confidence_interval.high:.3f}]")
```

### C. 样本量合理性说明

**功效分析 (Power Analysis)**:
- 预期效应量: r = 0.75 (大效应)
- 显著性水平: α = 0.05
- 统计功效: 1 - β = 0.80
- 所需样本量: N ≈ 30 (相关系数检验)

**实际选择 N = 200 的理由**:
1. 覆盖多样本类型 (正常/边界/堆砌)
2. 支持分层分析 (叙事主题/长度)
3. 预留 10% 无效样本剔除空间
4. 支持维度级效度分析 (6 维度 × 200 样本 = 1200 数据点)
