## 执行摘要

本实验设计形成**五层验证体系**，构建完整证据链：

```
L1a → Mock 测试      → 算法是否按设计运行？         → ✅ 已完成 (V4)
L1b → Benchmark 验证  → 评分与人工标注一致性如何？    → 🟡 设计中
L1c → 事件边界检测    → 事件提取准确率如何？         → 🟡 设计中
L2  → Pilot RCT       → 干预是否有效？              → 🟡 设计中
L3  → A/B 测试        → 元记忆策略是否有增量价值？   → 🟡 设计中
```

**核心设计原则**:
- **变量控制**: 四层控制矩阵（受试者→干预→评估→数据）+ 混淆变量显式建模
- **对比组设计**: 主动对照控制期望效应，多基线排除混淆，嵌套设计检验机制
- **评估指标**: 所有指标操作化定义 + 测量工具 + 评分规范 + 临床截断值

|------|--------|----------|----------|----------|

#### 1.3.3 干预内容标准化

**干预组 (CittaVerse) 标准流程**:
```
1. 签到与情绪检查 (2 分钟)
   - "您今天心情怎么样？" (0-10 评分)
   - "最近有什么新鲜事吗？" (暖场)

2. AI 引导回忆特定主题 (15-20 分钟)
   - 主题库: 童年/上学/工作/恋爱/家庭/成就/挑战 (轮换)
   - 引导语: 标准化提示库 (100% 使用，禁止即兴发挥)
   - ASR: Dolphin 方言引擎 v1.2 实时转写

3. 六维评分 (后台自动)
   - 算法版本: v0.6
   - 输出: 六维雷达图 + 综合分

4. 反馈与总结 (5 分钟)
   - 展示: 雷达图 (仅综合分 + 1 个最低维度)
   - 建议: 1-2 条标准化建议
   - 元记忆策略: 50% 随机会话触发 (系统日志记录)

5. 情感安全检查
   - 会话后评分: 0-10 (≥7.0 达标)
   - 关键词检测: 自伤/自杀→转人工
```

**对照组 (Active Control) 标准流程**:
```
1. 签到与情绪检查 (2 分钟)
   - 同干预组

2. 结构化认知训练 (25-30 分钟)
   - 记忆游戏: 图片回忆、数字广度、词语配对
   - 注意力任务: Stroop 任务、视觉搜索
   - 难度自适应: 正确率>80% 升级

3. 反馈与总结 (5 分钟)
   - 展示: 游戏得分 + 进步曲线
   - 鼓励: 标准化积极反馈

4. 情感安全检查
   - 同干预组
```

**关键区别**:
| 维度 | 干预组 | 对照组 |
|------|--------|--------|
| 核心成分 | 自传体叙事 + 元记忆 | 认知训练 |
| 回忆成分 | 有 | 无 |
| 叙事成分 | 有 | 无 |
| 社交互动 | 有 (AI 对话) | 有限 (游戏界面) |
| 接触时间 | 匹配 | 匹配 |
| 期望效应 | 匹配 | 匹配 |

#### 1.3.4 干预保真度监测

| 维度 | 监测方式 | 频率 | 达标标准 | 纠偏措施 |
|------|----------|------|----------|----------|

| 盲法类型 | 对象 | 控制方式 | 检验方法 |
|----------|------|----------|----------|

| 开放 | 干预执行者 | 无法盲 | — |

**盲法检验协议**:
```
在 8 周评估完成后，询问评估者:
"您认为该受试者属于哪一组？(干预组/对照组/不确定)"

计算:
- 正确率 = 正确猜测数 / 总猜测数
- 期望: 正确率 ≈ 50% (随机水平)
- 警戒线: 正确率 >70% → 盲法可能破坏
- 处理: 敏感性分析 (排除盲法破坏样本)
```

### 1.5 数据层面 (Data Level)

#### 1.5.1 数据录入质控

| 控制点 | 方法 | 频率 | 标准 |
|--------|------|------|------|
| 双人独立录入 | 两人独立录入同一数据 | 持续 | 100% 数据 |
| 差异比对 | 系统自动比对 | 每次录入后 | 差异>5% 触发重核 |

| 一致性检查 | 跨字段逻辑校验 | 每日 | 矛盾值标记 |

#### 1.5.2 缺失数据处理策略

| 缺失类型 | 缺失率 | 处理方法 | 假设 |
|----------|--------|----------|------|
| 基线缺失 | <5% | 多重插补 (m=20, PMM) | MAR |
| 随访缺失 | <20% | LMM 自动处理 | MAR |
| 随访缺失 | 20-30% | 模式混合模型 (PMM) | MNAR 检验 |
| 随访缺失 | >30% | Worst-case 分析 + 明确报告局限 | MNAR |

**MAR 假设检验**:
- Little's MCAR test (p>0.05 支持 MAR)
- 缺失模式分析: 比较完成者 vs 未完成者的基线特征
- 敏感性分析: δ-adjustment (假设缺失者比完成者差 0.5 SD)

#### 1.5.3 数据锁定流程

```
步骤 1: 所有数据录入完成 → 数据管理员确认
步骤 2: 离群值审查 (±3 SD 或临床不合理值) → PI 确认处理方式
步骤 3: 协议偏差记录完成 → 研究协调员确认
步骤 4: 数据库锁定 (REDCap 锁定功能) → PI 签名
步骤 5: 审计追踪保存 → 永久存档

锁定后:
- 禁止任何修改
- 如确需修改，需走偏差流程并记录
```

### 2.2 Pilot RCT 对比组 (L2)

#### 2.2.1 组别定义

| 组别 | 干预内容 | 理论依据 | 预期效应量 (d) |
|------|----------|----------|----------------|
| **干预组** | AI 引导回忆 + 六维评分反馈 + 元记忆策略 (50%) | 自传体记忆理论 + 元记忆训练 | 0.50-0.65 |
| **主动对照组** | 结构化认知训练 (记忆游戏 + 注意力任务) | 控制期望效应 + 接触时间 | 0.20-0.30 (练习效应) |

#### 2.2.2 为什么选择主动对照而非等待对照？

| 考量 | 主动对照 | 等待对照 |
|------|----------|----------|
| **伦理** | ✅ 对照组也有益干预 | ❌ 对照组无干预 |
| **期望效应控制** | ✅ 两组期望匹配 | ❌ 干预组期望更高 |
| **接触时间控制** | ✅ 两组时间匹配 | ❌ 接触时间不同 |
| **效应量估计** | ⚠️ 保守估计 (更严格) | ⚠️ 高估风险 |
| **Pilot 目的** | ✅ 评估可行性 + 初步疗效 | ❌ 仅评估可行性 |

**决策**: Pilot 阶段采用主动对照，伦理更优，效应量估计更保守，为后续确证性 RCT 留有余地。

#### 2.2.3 为什么对照组不含叙事成分？

**核心机制假设**:
```
叙事质量改善 → 认知功能改善
      ↑
   中介变量
```

**如对照组含叙事**:
- ❌ 无法隔离叙事这一核心机制
- ❌ 组间差异可能来自其他成分 (如评分反馈)
- ❌ 无法回答"叙事是否有效"这一核心问题

**对照组含认知训练的理由**:
- ✅ 控制"认知刺激"这一混淆变量
- ✅ 排除"任何认知活动都有效"的替代解释
- ✅ 更符合临床实践 (对照组也有益)

### 2.3 A/B 测试对比组 (L3, 嵌套于干预组)

#### 2.3.1 组别定义

| 组别 | n | 干预内容 | 元记忆策略 | 假设 |
|------|---|----------|------------|------|
| **A (标准)** | 20 | 标准 AI 引导 | 无 | 基线改善 |
| **B (增强)** | 20 | 标准 AI 引导 | 基于六维评分动态触发 | 改善斜率更陡 |

#### 2.3.2 元记忆策略触发规则

```python
# 伪代码
def trigger_metamemory_strategy(session_data):
    """
    基于六维评分动态触发元记忆策略
    """
    strategies = {
        '时间连贯性': '时间锚定 ("那大概是什么年代？您当时多大？")',
        '事件丰富度': '感官细节 ("那天的天气/声音/味道还记得吗？")',
        '因果连贯性': '因果追问 ("后来为什么会那样？")',
        '情感深度': '情感探索 ("您当时心里什么感觉？")',
        '自我认同': '自我连接 ("这件事对您后来的人生有什么影响？")',
        '信息密度': '重点聚焦 ("能再多说说那个关键时刻吗？")'
    }
    
    # 找出得分最低的维度 (<60 分)
    low_dimensions = [d for d, score in session_data.scores.items() if score < 60]
    
    # 本会话未触发过的策略
    available = [d for d in low_dimensions if d not in session_data.triggered_this_session]
    
    # 最多触发 2 个策略/会话
    to_trigger = available[:2]
    
    return [strategies[d] for d in to_trigger]
```

#### 2.3.3 主要假设

- **H1**: B 组六维评分改善斜率 > A 组 (β>0, p<0.05)
- **H2**: B 组低维度 (baseline<60) 改善幅度 > A 组
- **H3**: B 组用户满意度 (SUS, NPS) > A 组

### 2.4 算法基线对比 (L1c, 事件边界检测)

#### 2.4.1 基线系统

| 基线 | 描述 | 预期 F1 | 对比目的 |
|------|------|--------|----------|
| **随机基线** | 随机选择 20% 句子为边界 | ~0.20 | 排除随机性 |
| **标点基线** | 段落首句 + 问号/感叹号后 | ~0.40 | 排除表面线索 |
| **时间词基线** | 含时间词的句子 ("那年"、"后来") | ~0.50 | 排除单一语言线索 |
| **v0.5 算法** | 话题转换检测 v1 (仅语义相似度) | ~0.65 | 量化版本迭代改进 |
| **v0.6 算法** (本实验) | 话题转换检测 v2 (多线索融合) | >0.75 (目标) | 验证当前算法 |

#### 2.4.2 v0.6 算法核心改进

```
v0.5 → v0.6 改进点:

1. 多线索融合 (权重可调):
   - 语义相似度下降 (权重 0.4)
   - 时间标记出现 (权重 0.2)
   - 地点标记出现 (权重 0.15)
   - 人物标记变化 (权重 0.15)
   - 话题词 TF-IDF 突变 (权重 0.1)

2. 上下文窗口优化:
   - v0.5: 固定 3 句窗口
   - v0.6: 自适应 2-5 句 (基于叙事节奏)

3. 后处理规则:
   - 最小事件长度: ≥2 句
   - 边界平滑: 连续边界合并
   - 容差评估: ±1 句计为 TP
```

**SUS 评分计算**:
```
奇数题 (1,3,5,7,9): 得分 = 用户评分 - 1
偶数题 (2,4,6,8,10): 得分 = 5 - 用户评分
SUS 总分 = 所有题目得分之和 × 2.5

解释:
- ≥68: Above average (成功标准)
- ≥80.3: A 级 (优秀)
- <68: Below average (需改进)
```

**NPS 计算**:
```
推荐意愿 (0-10 分):
- 9-10 分: Promoters (推荐者)
- 7-8 分: Passives (被动者)
- 0-6 分: Detractors (贬损者)

NPS = Promoters% - Detractors%
范围: -100 到 +100
成功标准: ≥30
```

| 指标 | 计算方法 | 目标值 | 依据 | 解释标准 |
|------|----------|--------|------|----------|
| **评估者间信度** | ICC(2,1) 每维度 + 综合分 | >0.75 | CheckEval 先例 | <0.50=Poor, 0.50-0.75=Moderate, 0.75-0.90=Good, >0.90=Excellent |
| **自动 vs 人工相关** | Pearson r 每维度 + 综合分 | >0.70 | Healthcare LLM-Judge (r=0.818) | <0.10=Negligible, 0.10-0.29=Small, 0.30-0.49=Medium, ≥0.50=Large |
| **事件检测 F1** | 自动事件边界 vs 人工标注 | >0.75 | Event Segmentation 2025 | — |
| **边界容差 F1** | 允许±1 句误差 | >0.80 | 事件边界模糊性 | — |
| **Bland-Altman 一致性限** | 95% LoA | ≤±15 分 | 临床可接受范围 | — |
| **事件数相关** | 自动 vs 人工 (Pearson r) | >0.80 | — | — |

**Bland-Altman 分析**:
```
目的: 评估自动评分与人工评分的一致性限

计算:
1. 差值 = 自动评分 - 人工评分
2. 均值差 (bias) = mean(差值)
3. 一致性限 (LoA) = 均值差 ± 1.96 × SD(差值)
4. 95% 数据点应落在 LoA 内

成功标准:
- 95% LoA ≤ ±15 分 (临床可接受)
- 均值差接近 0 (无系统偏差)
- 差值与均值无相关 (无比例偏差)
```

- >30%: Worst-case 敏感性分析 (MNAR)

### 6.3 探索性分析

**叙事评分纵向模型**:

```r
model_narrative <- lmer(
  narrative_score ~ group * session_num * dimension + (1 | subject_id),
  data = narrative_data
)
```

```r
model_ab <- lmer(
  narrative_score ~ condition * session_num + (1 + session_num | subject_id),
```
