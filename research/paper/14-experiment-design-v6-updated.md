## 1. 变量控制矩阵 (Variable Control Matrix)

### 1.1 控制策略总览

采用四层控制矩阵 + 抗堆砌专项控制，系统化管理混淆变量：

| 控制层级 | 控制对象 | 主要方法 | 负责人 | 容差标准 | 监测频率 |
|----------|----------|----------|--------|----------|----------|
| **受试者层** | 纳入/排除/协变量 | 标准化筛查 + 随机化分层 | 研究协调员 | 纳入标准±0 | 每受试者 |
| **干预层** | 剂量/内容/保真度 | 系统日志 + 10% 人工抽检 | 干预执行者 | 时长 25-45min | 每会话 |
| **评估层** | 评估者/时点/盲法 | 统一培训 + 盲法检验 | 评估主管 | ICC>0.90 | 每周 |
| **数据层** | 录入/缺失/锁定 | 双人录入 + 统计调整 | 数据管理员 | 差异>5% 重核 | 每日 |
| **抗堆砌层** | 关键词堆砌检测 | L0 密度检测 + L1 语义一致性 | 系统自动 | 堆砌率<2% | 实时 |

#### 1.3.3 干预内容标准化

**干预组 (CittaVerse) 标准流程** (30-40 分钟):

```
1. 签到与情绪检查 (2 分钟)
   - "您今天心情怎么样？" (0-10 评分)
   - "最近有什么新鲜事吗？" (暖场)

2. AI 引导回忆特定主题 (15-20 分钟)
   - 主题库：童年/上学/工作/恋爱/家庭/成就/挑战 (轮换)
   - 引导语：标准化提示库 (100% 使用，禁止即兴发挥)
   - ASR: Dolphin 方言引擎 v1.2 实时转写

3. 六维评分 (后台自动)
   - 算法版本：v0.6 (L0+L1 融合)
   - L1 触发条件：置信度<0.6 或 边界案例
   - L1 触发率目标：20% ± 5%
   - 输出：六维雷达图 + 综合分 + 置信度

4. 反馈与总结 (5 分钟)
   - 展示：雷达图 (仅综合分 + 1 个最低维度)
   - 建议：1-2 条标准化建议
   - 元记忆策略：50% 随机会话触发 (系统日志记录)

5. 情感安全检查
   - 会话后评分：0-10 (≥7.0 达标)
   - 关键词检测：自伤/自杀→转人工

6. 抗堆砌检测 (新增)
   - 信息密度异常检测：>3 SD 标记
   - 情感 - 事件一致性检查：不匹配标记
   - 堆砌样本处理：评分打折 + 人工复核
```

**对照组 (Active Control) 标准流程** (30-40 分钟):

```
1. 签到与情绪检查 (2 分钟)
   - 同干预组

2. 结构化认知训练 (25-30 分钟)
   - 记忆游戏：图片回忆、数字广度、词语配对
   - 注意力任务：Stroop 任务、视觉搜索
   - 难度自适应：正确率>80% 升级

3. 反馈与总结 (5 分钟)
   - 展示：游戏得分 + 进步曲线
   - 鼓励：标准化积极反馈

4. 情感安全检查
   - 同干预组
```

**关键区别**:

| 维度 | 干预组 | 对照组 | 控制目的 |
|------|--------|--------|----------|
| 核心成分 | 自传体叙事 + 元记忆 | 认知训练 | 隔离叙事机制 |
| 回忆成分 | 有 | 无 | 检验自传体记忆理论 |
| 叙事成分 | 有 | 无 | 检验叙事质量→认知改善路径 |
| 社交互动 | 有 (AI 对话) | 有限 (游戏界面) | 部分匹配 |
| 接触时间 | 匹配 | 匹配 | 排除接触时间混淆 |
| 期望效应 | 匹配 | 匹配 | 排除安慰剂效应 |
| **评分反馈** | **有 (六维雷达图)** | **无 (仅游戏得分)** | **隔离评分反馈效应** |

### 1.6 抗堆砌专项控制 (Anti-Gaming Control) — 新增

基于 2026-04-03 L0 鲁棒性测试发现，新增堆砌检测与控制：

#### 1.6.1 堆砌类型定义

| 堆砌类型 | 检测信号 | 阈值 | 处理策略 |
|----------|----------|------|----------|
| **C1 感官堆砌** | 感官词密度 >3 SD | >6 词/100 字 | L1 仲裁 + 评分打折 |
| **C2 实体堆砌** | 外部实体密度 >3 SD | >8 实体/100 字 | L1 仲裁 + 评分打折 |
| **C4 情感堆砌** | 情感词密度 >3 SD + 效价不匹配 | >5 词/100 字 | L1 仲裁 + 评分打折 |
| **混合堆砌** | 多维度同时异常 | ≥2 维度异常 | L1 仲裁 + 人工复核 |

#### 1.6.2 检测算法

```python
def detect_gaming(text, scores):
    """
    检测关键词堆砌 (Reward Hacking)
    """
    gaming_signals = {
        'sensory_density': count_sensory_words(text) / len(text) * 100,
        'entity_density': count_external_entities(text) / len(text) * 100,
        'emotion_density': count_emotion_words(text) / len(text) * 100,
        'emotion_event_mismatch': check_emotion_event_consistency(text, scores)
    }
    
    thresholds = {
        'sensory_density': 6.0,      # 词/100 字
        'entity_density': 8.0,       # 实体/100 字
        'emotion_density': 5.0,      # 词/100 字
        'emotion_event_mismatch': True  # 布尔
    }
    
    detected = []
    for signal, value in gaming_signals.items():
        if signal == 'emotion_event_mismatch':
            if value:
                detected.append(signal)
        else:
            if value > thresholds[signal]:
                detected.append(signal)
    
    return {
        'is_gaming': len(detected) >= 1,
        'gaming_types': detected,
        'severity': len(detected)  # 1=轻度，2+=重度
    }
```

#### 1.6.3 处理策略

| 严重程度 | 处理方式 | 评分调整 |
|----------|----------|----------|
| **无堆砌** | 正常评分 | 无调整 |
| **轻度 (1 维度)** | L1 仲裁 + 警告标记 | 综合分 × 0.95 |
| **中度 (2 维度)** | L1 仲裁 + 人工复核 | 综合分 × 0.85 |
| **重度 (3+ 维度)** | 人工复核 + 可能排除 | 综合分 × 0.70 或排除 |

### 2.2 Pilot RCT 对比组 (L2)

#### 2.2.1 组别定义

| 组别 | 干预内容 | 理论依据 | 预期效应量 (d) |
|------|----------|----------|----------------|
| **干预组** | AI 引导回忆 + 六维评分反馈 + 元记忆策略 (50%) | 自传体记忆理论 + 元记忆训练 | 0.50-0.65 |
| **主动对照组** | 结构化认知训练 (记忆游戏 + 注意力任务) | 控制期望效应 + 接触时间 | 0.20-0.30 (练习效应) |

#### 2.2.2 为什么选择主动对照而非等待对照？

| 考量 | 主动对照 | 等待对照 | 决策 |
|------|----------|----------|------|
| **伦理** | ✅ 对照组也有益干预 | ❌ 对照组无干预 | 主动对照更优 |
| **期望效应控制** | ✅ 两组期望匹配 | ❌ 干预组期望更高 | 主动对照更优 |
| **接触时间控制** | ✅ 两组时间匹配 | ❌ 接触时间不同 | 主动对照更优 |
| **效应量估计** | ⚠️ 保守估计 (更严格) | ⚠️ 高估风险 | 主动对照更严格 |
| **Pilot 目的** | ✅ 评估可行性 + 初步疗效 | ❌ 仅评估可行性 | 主动对照更全面 |

**决策理由**: Pilot 阶段采用主动对照，伦理更优，效应量估计更保守，为后续确证性 RCT 留有余地。

#### 2.2.3 为什么对照组不含叙事成分？

**核心机制假设**:
```
叙事质量改善 → 认知功能改善
      ↑
   中介变量
```

**如对照组含叙事**:
- ❌ 无法隔离叙事这一核心机制
- ❌ 组间差异可能来自其他成分 (如评分反馈)
- ❌ 无法回答"叙事是否有效"这一核心问题

**对照组含认知训练的理由**:
- ✅ 控制"认知刺激"这一混淆变量
- ✅ 排除"任何认知活动都有效"的替代解释
- ✅ 更符合临床实践 (对照组也有益)

#### 2.3.2 元记忆策略触发规则

基于六维评分动态触发元记忆策略 (伪代码):

```python
def trigger_metamemory_strategy(session_data):
    """
    基于六维评分动态触发元记忆策略
    """
    strategies = {
        '时间连贯性': '时间锚定 ("那大概是什么年代？您当时多大？")',
        '事件丰富度': '感官细节 ("那天的天气/声音/味道还记得吗？")',
        '因果连贯性': '因果追问 ("后来为什么会那样？")',
        '情感深度': '情感探索 ("您当时心里什么感觉？")',
        '自我认同': '自我连接 ("这件事对您后来的人生有什么影响？")',
        '信息密度': '重点聚焦 ("能再多说说那个关键时刻吗？")'
    }
    
    # 找出得分最低的维度 (<60 分)
    low_dimensions = [d for d, score in session_data.scores.items() if score < 60]
    
    # 本会话未触发过的策略
    available = [d for d in low_dimensions if d not in session_data.triggered_this_session]
    
    # 最多触发 2 个策略/会话
    to_trigger = available[:2]
    
    return [strategies[d] for d in to_trigger]
```

- **H3**: B 组用户满意度 (SUS, NPS) > A 组

### 4.5 样本量计算

```
公式：N_per_group = (z_α/2 + z_β)² × 2 × σ² / δ²

参数:
├─ z_α/2 = 1.96 (α=0.05, 双尾)
├─ z_β = 0.84 (β=0.20, 80% power)
├─ δ = 2.0 (MoCA 组间差异，基于 Pu 2025 meta)
└─ σ = 3.0 (MoCA SD, 文献先例)

计算：N = (1.96+0.84)² × 2 × 9 / 4 = 29.4 → 30
考虑 20% 脱落：30 / 0.8 = 37.5 → 每组 40 人，总 N=80

注：Pilot 主要评估可行性，疗效为探索性，样本量以满足效应量估计精度为主
```

**EXP-001 样本量计算**:
```
目标：检测 Δr = 0.10 的相关系数差异 (r=0.65 → r=0.75)

使用 G*Power 3.1:
- Test family: t tests
- Statistical test: Correlation: Difference from constant
- Type of power analysis: A priori
- Effect size: q = 0.10 (Fisher's z transformation)
- α err prob: 0.05
- Power (1-β err prob): 0.80

输出：N = 194 → 200 (考虑分层需求)
```

| 时长 | 系统自动记录 | 25-45 分钟/次占比≥90% | 超范围会话人工审核 |
| 频率 | 系统预约记录 | 每周 2 次，间隔≥2 天 | 缺席电话随访 |
| 内容 | AI 提示语日志 | 100% 使用标准库 | 偏差会话标记排除 |
| 质量 | 10% 人工抽检 | 评分一致率>90% | 评估者复训 |
| 脱落 | 连续 2 次缺席 | 脱落率<30% | 加强随访/补偿 |
| 技术故障 | 系统错误日志 | ≤5% | 技术团队待命 |
| **L1 触发** | **系统日志** | **20% ± 5%** | **调整置信度阈值** |
| **堆砌检出** | **系统日志** | **<2%** | **>2% 触发人工复核** |

### 5.2 评估者盲法检验

```
在 8 周评估完成后，询问评估者:
"您认为该受试者属于哪一组？(干预组/对照组/不确定)"

计算:
├─ 正确率 = 正确猜测数 / 总猜测数
├─ 期望：正确率 ≈ 50% (随机水平)
└─ 如正确率 >70%: 盲法可能破坏，需敏感性分析

处理:
├─ 正确率 50-70%: 可接受，记录
└─ 正确率 >70%: 排除该评估者评估的样本，重新分析
```

### 5.3 数据质控

| 检查点 | 方法 | 频率 |
|--------|------|------|
| 录入错误 | 双人独立录入，差异>5% 重核 | 持续 |
| 离群值 | ±3 SD 或临床不合理值标记 | 数据锁定前 |
| 缺失模式 | Little's MCAR test | 分析前 |
| 协议偏差 | 记录所有偏离方案的情况 | 持续 |
| **堆砌标记** | **系统自动检测 + 人工复核** | **每会话** |
