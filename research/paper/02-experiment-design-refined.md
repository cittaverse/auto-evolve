### 1.2 待完成: Benchmark 数据集 (变量控制强化版)

**状态**: 🟡 设计中 (GEO #63 计划)

**核心设计**:

```
目标: 创建中文叙事评估的 gold standard 数据集

样本:
├─ 总样本量: N = 100 段叙事
├─ 来源: Pilot RCT 受试者 (MCI, n=60) + 健康对照 (n=40)
├─ 年龄分布: 60-85 岁 (每 5 岁一层，确保代表性)
├─ 性别: 1:1 平衡
├─ 教育水平: 小学/初中/高中/大学 (各 25%)
└─ 叙事主题: 童年/青年/中年/晚年 (各 25%)

变量控制:
├─ 录音设备: 统一使用 iPhone 内置麦克风 (距离 30cm, 安静环境)
├─ 录音时长: 3-5 分钟/段 (截断超出部分)
├─ ASR 引擎: 统一使用 Dolphin 方言 ASR (v1.2)
├─ 转写质量: CER < 15% (人工校对达标)
├─ 文本格式: 纯文本，保留原始标点，不人工分段
└─ 元数据: 匿名化 PID, 年龄组，性别，教育，主题
```

**标注协议 (Checklist-Based 范式)**:

基于 CheckEval (Kim et al., 2024) 和 Healthcare LLM-Judge (Velez et al., 2025) 的方法论，采用**二元 checklist 分解**而非直接 Likert 评分，以提升评估者间一致性。

| 维度 | Checklist Items (示例) | 计分方式 |
|------|------------------------|----------|
| **事件丰富度** | 1. 包含≥3 个离散事件<br>2. 每个事件有明确时间/地点<br>3. 事件之间有清晰边界<br>4. 包含至少 1 个高潮事件<br>5. 事件密度适中 (不拥挤不稀疏) | Y=1, N=0<br>总分 0-5 → ×20 = 0-100 |
| **时间连贯性** | 1. 使用≥3 个时间标记词<br>2. 时间顺序清晰无矛盾<br>3. 有明确的时间起点和终点<br>4. 时间跨度合理<br>5. 时间跳跃有过渡说明 | Y=1, N=0<br>总分 0-5 → ×20 = 0-100 |
| **因果连贯性** | 1. 包含≥2 个因果关系<br>2. 因果连接词使用恰当<br>3. 事件结果有解释<br>4. 动机/原因有说明<br>5. 因果链无明显断裂 | Y=1, N=0<br>总分 0-5 → ×20 = 0-100 |
| **情感深度** | 1. 包含≥3 个情感词<br>2. 情感与事件有因果关联<br>3. 描述了身体感受变化<br>4. 情感有强度变化<br>5. 有情感反思/评价 | Y=1, N=0<br>总分 0-5 → ×20 = 0-100 |
| **自我认同** | 1. 使用≥5 次自我指称 (我/自己)<br>2. 有自我特质描述<br>3. 有自我变化叙述<br>4. 事件与自我概念关联<br>5. 有人生意义反思 | Y=1, N=0<br>总分 0-5 → ×20 = 0-100 |
| **信息密度** | 1. 中心信息占比 40-60%<br>2. 中心事件有细节展开<br>3. 周边信息不喧宾夺主<br>4. 信息分布有节奏感<br>5. 无冗余重复 | Y=1, N=0<br>总分 0-5 → ×20 = 0-100 |

**评估者培训**:

```
阶段 1 (2 小时): 阅读六维框架定义 + 评分指南 + 10 段练习
阶段 2 (2 小时): 2 名评估者独立评分 10 段 → 讨论差异 → 达成共识
阶段 3 (1 小时): 独立评分 20 段校准叙事
达标标准: 校准叙事 ICC > 0.80
不达标处理: 追加讨论 + 重新校准 (最多 3 轮)
```

**目标指标**:

| 指标 | 计算方法 | 目标值 | 依据 |
|------|----------|--------|------|
| 评估者间信度 | ICC(2,1) | > 0.75 | CheckEval 先例 (提升 0.45) |
| 自动 vs 人工相关 | Pearson r (每维度) | > 0.70 | Healthcare LLM-Judge (r=0.818) |
| 事件检测 F1 | 自动事件边界 vs 人工标注 | > 0.75 | Event Segmentation 2025 先例 |
| Bland-Altman 一致性限 | 95% LoA | ≤ ±15 分 | 临床可接受范围 |

### 1.3 待完成: v0.5 vs v0.6 对比 (算法迭代验证)

**目的**: 量化事件边界检测 v2 的改进

**变量控制**:

| 变量类型 | 变量名 | 控制方式 |
|----------|--------|----------|
| **自变量** | 算法版本 | v0.5 (基线) vs v0.6 (话题转换感知) |
| **因变量** | 事件检测 F1 | vs 人工标注 gold standard |
| **因变量** | 维度评分相关 | vs 人工评分 (Pearson r) |
| **控制变量** | 测试集 | 同一组叙事 (n=100) |
| **控制变量** | 人工标注 | 同一批评估者，盲法评分 |
| **控制变量** | 预处理 | 相同 ASR 引擎，相同文本清洗 |

**设计**:
```
同一组叙事 (n=100) 
    ↓
    ├─→ v0.5 评分 → 事件提取 → 六维评分
    └─→ v0.6 评分 → 事件提取 → 六维评分
                    ↓
    比较:
    ├─ 事件提取数量 (v0.6 ≥ v0.5 预期)
    ├─ 事件边界准确率 (vs 人工标注, F1)
    ├─ 中心/周边分类准确率 (vs 人工标注)
    └─ 综合评分与人工的相关性 (Pearson r)
    
假设:
├─ H1: v0.6 事件检测 F1 显著高于 v0.5 (ΔF1 > 0.10)
├─ H2: v0.6 维度评分与人工相关更高 (Δr > 0.05)
└─ H3: v0.6 在长叙事 (>500 字) 上改进更明显
```

**统计检验**:
- 事件检测 F1 差异: McNemar 检验 (配对设计)
- 相关系数差异: Fisher's z 检验
- 效应量: Cohen's d (组内设计)

## 实验 2: Pilot RCT (L2)

### 2.1 研究设计 (强化变量控制)

**设计类型**: 多中心、随机、对照、单盲 Pilot RCT

| 参数 | 值 | 变量控制说明 |
|------|---|--------------|
| **样本量** | N=60-80 (30-40/组) | 基于效应量 d=0.65, power=80% |
| **干预时长** | 8 周 (每周 2 次) | 固定频率，允许±3 天弹性 |
| **对照** | 主动对照 (结构化认知训练) | 控制期望效应和接触时间 |
| **盲法** | 单盲 (评估者盲) | 评估者不知分组，受试者/干预者不盲 |
| **随机化** | 1:1 区组随机 (block size=4) | 按中心分层随机 |
| **中心数** | 2-3 (杭州社区 + 记忆门诊) | 中心作为协变量纳入模型 |
| **分配隐藏** | REDCap 随机化模块 | 入组后实时分配，不可预测 |

### 2.2 受试者 (纳入排除标准化)

**纳入标准**:
- 年龄 60-85 岁 (以身份证为准)
- MCI: MoCA 18-25 (中文版，教育校正)
- 普通话或当地方言 (可交流)
- 矫正后可正常交流 (视力/听力)
- 知情同意 (本人 + 家属)

**排除标准**:
- 重度痴呆 (MoCA <18)
- 严重精神疾病 (精神病性症状，双相，自杀风险)
- 近 3 月参加其他 RCT
- 严重躯体疾病 (预期寿命<6 月)
- 无法配合完成评估 (严重听力/视力障碍)

**招募流程 (变量控制)**:

```
社区宣传/门诊转介
    ↓
电话初筛 (年龄，MoCA 自报，排除标准)
    ↓
现场筛查 (知情同意，MoCA 测试，纳入确认)
    ↓
基线评估 (所有终点指标)
    ↓
随机化 (REDCap, 区组随机，分配隐藏)
    ↓
    ├─→ 干预组 (CittaVerse)
    └─→ 对照组 (主动对照)
```

**样本量计算依据**:

```
公式: N_per_group = (z_α/2 + z_β)² × 2 × σ² / δ²

参数:
├─ z_α/2 = 1.96 (α=0.05, 双尾)
├─ z_β = 0.84 (β=0.20, 80% power)
├─ δ = 2.0 (MoCA 组间差异，基于 Pu 2025 数字 RT meta)
└─ σ = 3.0 (MoCA SD, 基于文献先例)

计算: N_per_group = (1.96 + 0.84)² × 2 × 9 / 4 = 29.4 → 30

考虑 20% 脱落: 30 / 0.8 = 37.5 → 每组 40 人，总 N=80
```

### 2.3 干预方案 (标准化操作程序)

**干预组 (CittaVerse)**:

```
每次会话标准流程 (30-40 分钟):

1. 签到与情绪检查 (2 分钟)
   - 当前情绪评分 (0-10)
   - 过去一周重大事件 (1-2 句)

2. AI 引导回忆特定主题 (15-20 分钟)
   - 主题库: 童年游戏/第一份工作/最难忘旅行/...
   - AI 提示语标准化 (见 Appendix A)
   - 元记忆策略 (随机 50% 会话触发，用于 A/B 测试)

3. ASR 实时转写 + 六维评分 (后台运行)
   - ASR: Dolphin 方言引擎
   - 评分: v0.6 算法
   - 输出: 六维分数 + 综合分 + 等级

4. 反馈与总结 (5 分钟)
   - 展示本次评分 (可视化雷达图)
   - AI 生成 1-2 条具体改进建议
   - 预约下次会话时间

5. 人类专家抽检 (10% 随机会话)
   - 质量审核
   - 情感风险筛查 (黄色/红色警报处理)
```

**对照组 (Active Control)**:

```
每次会话标准流程 (30-40 分钟):

1. 签到与情绪检查 (2 分钟)
   - 同干预组

2. 结构化认知训练 (25-30 分钟)
   - 记忆游戏: 数字广度，配对记忆，位置回忆
   - 注意力任务: Stroop, 视觉搜索
   - 难度自适应 (正确率>80% 升级)
   - 不含叙事/回忆成分 (关键区别)

3. 反馈与总结 (5 分钟)
   - 展示游戏得分
   - 鼓励性反馈 (标准化话术)
   - 预约下次会话时间

4. 人类专家抽检 (10% 随机会话)
   - 同干预组
```

**干预保真度控制**:

| 维度 | 控制方式 | 达标标准 |
|------|----------|----------|
| **时长** | 系统自动记录 | 25-45 分钟/次 |
| **频率** | 系统预约提醒 | 每周 2 次，间隔≥2 天 |
| **内容** | AI 提示语标准化 | 100% 使用标准库 |
| **质量** | 10% 人工抽检 | 评分一致率>90% |
| **脱落** | 连续 2 次缺席电话随访 | 脱落率<30% |

## 实验 3: 评分标准验证 (L3)

### 3.1 设计 (强化版)

**目的**: 建立 CittaVerse 六维自动评分与人类专家评分的收敛效度

**样本**: 从 Pilot RCT 中随机抽取 100 段叙事 (干预组 + 对照组混合)

**抽样策略**:
```
分层随机抽样:
├─ 按组别分层: 干预组 60 段，对照组 40 段
├─ 按时点分层: 基线 25 段，4 周 25 段，8 周 50 段
├─ 按质量分层: S/A/B/C/D 各 20 段 (基于自动评分预分层)
└─ 按主题分层: 童年/青年/中年/晚年各 25 段

样本量依据:
├─ 目标: 检测 r=0.70 的相关性 (α=0.05, power=90%)
├─ 公式: N = (z_α/2 + z_β)² / (0.5 × ln((1+r)/(1-r)))² + 3
├─ 计算: N = (1.96 + 1.28)² / (0.5 × ln(1.7/0.3))² + 3 = 84.6 → 85
└─ 考虑 15% 无效样本: 85 / 0.85 = 100 段
```

**评估者**: 2 名训练过的评估者 (心理学/语言学背景)

**评估者资质**:
- 学历: 心理学/语言学硕士及以上
- 经验: ≥2 年叙事研究或临床评估经验
- 培训: 完成 3 阶段培训协议 (见 1.2)
- 盲法: 不知自动评分结果，不知受试者分组

### 3.2 评估流程 (标准化)

```
100 段叙事文本 (匿名化)
    ↓
导入 REDCap 评估模块 (随机顺序呈现)
    ↓
评估者 A 独立评分 (6 维度 × checklist × 0-100)
    ↓
评估者 B 独立评分 (6 维度 × checklist × 0-100)
    ↓
计算:
├─ 评估者间信度: ICC(2,1) 每维度 + 综合分
├─ 自动 vs 人工 A 相关: Pearson r 每维度 + 综合分
├─ 自动 vs 人工 B 相关: Pearson r 每维度 + 综合分
└─ 自动 vs 人工平均相关: r_mean = (r_A + r_B) / 2
    ↓
Bland-Altman 分析:
├─ 计算差值: d = 自动评分 - 人工平均分
├─ 计算均值: mean_d
├─ 计算 SD: SD_d
└─ 95% 一致性限: mean_d ± 1.96 × SD_d
```

### 3.3 统计分析 (强化版)

| 分析 | 方法 | 目标 | 成功标准 |
|------|------|------|----------|
| **评估者间信度** | ICC(2,1) (双向随机，单次测量) | 每维度 + 综合分 | ICC > 0.75 (substantial) |
| **自动 vs 人工相关** | Pearson r (双尾) | 每维度 + 综合分 | r > 0.70 (strong) |
| **相关性差异检验** | Fisher's z 检验 | 6 维度间 r 差异 | 无显著差异 (p>0.05) |
| **一致性分析** | Bland-Altman plot + 95% LoA | 系统偏差 + 一致性限 | mean_d ≈ 0, LoA ≤ ±15 分 |
| **区分效度** | 独立样本 t 检验 (MCI vs 健康) | 自动评分能否区分组别 | d > 0.50 (medium) |
| **变化敏感性** | 配对 t 检验 (干预前后) | 自动评分能否检测变化 | d > 0.30 (small-medium) |
| **内部一致性** | Cronbach's α (6 维度) | 维度间相关性 | α = 0.70-0.90 (理想) |

**ICC 解释标准** (Koo & Li, 2016):
- < 0.50: Poor
- 0.50-0.75: Moderate
- 0.75-0.90: Good
- > 0.90: Excellent

**相关系数解释标准** (Cohen, 1988):
- < 0.10: Negligible
- 0.10-0.29: Small
- 0.30-0.49: Medium
- ≥ 0.50: Large

**Bland-Altman 解释**:
- mean_d 显著≠0: 存在系统偏差 (自动评分系统性偏高/偏低)
- LoA 过宽 (>30 分): 个体层面一致性不足，仅适合群体比较
- 散点呈漏斗形: 差异随评分大小变化 (异方差)

## 实验 4: A/B 测试 — 元记忆增强 (探索性)

### 4.1 设计 (嵌套 RCT)

**嵌套在 Pilot RCT 干预组内** (n=30-40):

| 组别 | n | 干预内容 | 区别 |
|------|---|----------|------|
| **A 组** (标准) | 15-20 | 标准 AI 引导 | 无元记忆策略 |
| **B 组** (增强) | 15-20 | 标准 AI 引导 + 元记忆策略提示 | 基于六维评分动态触发 |

**随机化**: 干预组内 1:1 随机 (独立于主 RCT 随机化)

**盲法**: 单盲 (评估者盲，受试者/干预者不盲)

### 4.2 元记忆策略 (操作化定义)

| 策略 | 触发条件 | 提示示例 | 理论依据 |
|------|----------|----------|----------|
| **时间锚定** | 时间连贯性 < 60 | "那大概是什么年代的事？您当时多大？" | Conway SMS 模型 |
| **感官细节** | 事件丰富度 < 60 | "那天的天气/声音/味道还记得吗？周围有什么？" | 情节记忆编码 |
| **因果追问** | 因果连贯性 < 60 | "后来为什么会那样？是什么导致了这个结果？" | 因果推理 |
| **情感探索** | 情感深度 < 60 | "回想起来，您当时心里什么感觉？现在呢？" | 情感记忆增强 |
| **自我连接** | 自我认同 < 60 | "这件事对您后来的人生有什么影响？您从中学到了什么？" | 叙事身份理论 |

**触发规则**:
```
每次会话后计算六维评分
    ↓
对每个维度:
├─ IF 维度分 < 60 AND 该维度策略本会话未触发
└─ THEN 下次会话中随机选择 1-2 个策略触发
    ↓
每会话最多触发 2 个策略 (避免认知过载)
策略触发顺序: 随机 (避免固定模式)
```

### 4.3 主要假设

| 假设 | 预测 | 检验方法 |
|------|------|----------|
| **H1 (主要)** | B 组六维评分改善斜率 > A 组 | 多层线性模型 (组别×会话交互) |
| **H2** | B 组会话完成率 > A 组 | χ² 检验 |
| **H3** | B 组满意度 (SUS) > A 组 | 独立样本 t 检验 |
| **H4** | B 组特定维度改善更明显 (与策略对应) | 维度×组别×时间三阶交互 |

### 4.4 分析方法

**多层线性模型 (HLM)**:

```
Level 1 (会话层):
  Narrative_Score_ij = β0j + β1j·Session_ij + ε_ij

Level 2 (个体层):
  β0j = γ00 + γ01·Condition_j + u0j
  β1j = γ10 + γ11·Condition_j + u1j

合并模型:
  Narrative_Score_ij = γ00 + γ01·Condition_j 
                     + γ10·Session_ij + γ11·(Condition×Session)_ij
                     + u0j + u1j·Session_ij + ε_ij

关键系数:
├─ γ11 = 组别×会话交互 → 改善速度差异 (主要检验)
├─ γ01 = 组别主效应 → 基线差异 (应不显著，验证随机化成功)
└─ γ10 = 会话主效应 → 整体改善趋势
```

**软件实现** (R):
```r
library(lme4)
model_ab <- lmer(
  narrative_score ~ condition * session_num + (1 + session_num | subject_id),
  data = ab_test_data
)
summary(model_ab)
anova(model_ab)  # 检验交互项显著性
```

**效应量计算**:
```r
library(effectsize)
cohens_d(model_ab, type = "d")  # 组间差异 d
```

## 实验 5: 事件边界检测验证 (新增，v2.0)

**目的**: 独立验证 v0.6 事件边界检测算法的性能

**设计依据**: Event Segmentation + Recall Assessment (arXiv:2502.13349) 方法论

### 5.1 样本

- **来源**: Benchmark 数据集 (实验 1.2) 的 100 段叙事
- **人工标注**: 2 名评估者独立标注事件边界
- **标注单位**: 句子级 (每个句子标注：新事件开始 / 事件继续)

### 5.2 标注协议

```
评估者阅读叙事，逐句判断:
├─ [新事件开始]: 时间/地点/人物/活动发生显著变化
├─ [事件继续]: 同一事件的延续
└─ [不确定]: 标记为模糊边界 (不计入最终分析)

标注工具: 自定义 Web 界面 (句子高亮 + 按钮点击)
一致性检验: 2 名评估者独立标注 20 段 → 计算 Cohen's κ
达标标准: κ > 0.70
```

### 5.3 评估指标

| 指标 | 计算方法 | 目标值 |
|------|----------|--------|
| **精确率 (Precision)** | TP / (TP + FP) | > 0.75 |
| **召回率 (Recall)** | TP / (TP + FN) | > 0.75 |
| **F1 分数** | 2 × P × R / (P + R) | > 0.75 |
| **边界容差 F1** | 允许±1 句误差 | > 0.80 |
| **事件数相关性** | 自动事件数 vs 人工事件数 (Pearson r) | > 0.80 |

**边界容差处理**:
- 自动检测边界在人工标注边界±1 句范围内 → 计为 TP
- 理由: 事件边界本身存在模糊性，人类评估者间也存在±1 句差异

### 5.4 对比基线

| 基线 | 描述 | 预期 F1 |
|------|------|--------|
| **随机基线** | 随机选择 20% 句子为边界 | ~0.20 |
| **标点基线** | 段落首句 + 问号/感叹号后 | ~0.40 |
| **时间词基线** | 含时间词的句子 | ~0.50 |
| **v0.5 算法** | 话题转换检测 v1 | ~0.65 (Mock 测试估计) |
| **v0.6 算法** | 话题转换检测 v2 (本实验) | > 0.75 (目标) |
