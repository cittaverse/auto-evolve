## 实验设计

### 7 个可消融组件

| ID | 组件 | 功能 | Fallback |
|----|------|------|----------|
| C1 | 情绪唤醒度检测 | 1-5 分情绪水平 | 固定=3 |
| C2 | 动态理想比例 | 根据唤醒度调整中心/外围比 | 固定 60/40 |
| C3 | LLM 事件提取 | LLM 提取并标注事件 | 规则启发式 |
| C4 | L0 六维评分 | 时间/地点/人物/感官/情感/连贯性 | 简化 3 维 |
| C5 | 多 Agent 评委 | 4 个独立评委 | 单一评委 |
| C6 | 投票加权 | 置信度加权 | 简单平均 |

### 12 种实验配置

**单组件消融 (8 种)**:
1. `full` — 完整系统 (基线)
2. `w/o_C1_arousal` — 关闭情绪唤醒度
3. `w/o_C2_dynamic_ratio` — 关闭动态比例
4. `w/o_C3_llm_event` — 关闭 LLM 事件提取

8. `w/o_C7_no_arbitration` — 无仲裁

**极端配置 (4 种)**:
9. `minimal` — 最小系统 (规则事件 + 简化评分 + 简单平均)

## 意外发现

### 1. 仲裁率 100%

**现象**: Full 配置下所有 50 条样本都触发仲裁

**根因**: Mock 中仲裁阈值设置为 15 分，4 个评委分数差异普遍超过此值

**建议**: 
- 调整阈值为 20-25 分
- 或重新设计评委一致性机制

### 2. 等级分布过于集中

**现象**: 所有配置的等级都集中在 C/D 级，无 S/A/B

**根因**: Mock 评分逻辑未充分模拟真实分数分布

**建议**: 
- 改进 Mock 评分函数
- 使用真实样本重新测试

### 3. 简化系统表现更优

**现象**: minimal/llm_only/rule_only 都比 full 表现更好
- 评分提升：+4.8~5.7 分
- 稳定性提升：std 从 4.22 降至 1.36 (3 倍)

**可能原因**:
1. Full 系统组件间交互产生冲突
2. 仲裁机制引入噪声 (100% 仲裁率)
3. Mock 中组件权重配置不合理

## 建议的 v0.6 架构

```
┌─────────────────────────────────────────────────────────┐
│                    L0/VSNC v0.6                         │
├─────────────────────────────────────────────────────────┤
│  输入 → LLM 事件提取 (C3) → 多 Agent 评委 (C5, 2-3 个)   │
```

- 评分稳定性提升 ~3 倍 (std: 4.22 → 1.36)

- 系统复杂度降低 (7 组件 → 4 组件)
- 延迟无明显变化
