## Bottom Line

CittaVerse 当前处于 **MVP 验证阶段**，核心假设（AI 辅助叙事评分可提升回忆治疗效果）尚未被真实用户数据证实。路线图应围绕"最快拿到用户反馈 → 验证核心假设 → 扩展变现"展开，而非追求功能完备性。

---

## 阶段定义

### 🟢 MVP (当前 → v0.3) — "能跑通一个完整用户循环"

**目标**: 1 个真实用户完成"语音输入 → 叙事评分 → 反馈报告"的完整闭环  
**预计时间**: 2026 Q1-Q2 (现在 ~ 6 月)  
**核心指标**: 首个完整用户循环完成

| 组件 | 当前状态 | MVP 需要 | Gap |
|------|----------|----------|-----|
| **Narrative Scorer** | v0.5 (规则引擎，6 维度) | 可对真实中文口述打分 | ⚠️ 未在真实用户口述上测试 |
| **语音输入** | ASR 选型完成，待 API Key | 基础中文 ASR (普通话) | 🔴 API Key 阻塞 >7 天 |
| **用户界面** | 无 | 最简微信小程序或 H5 | 🔴 未开始 |
| **反馈报告** | 评分 JSON 输出 | 可视化报告 (PDF/H5) | 🟡 模板设计完成，渲染未实现 |
| **数据存储** | 本地文件 | 基础云存储 (阿里云 OSS) | 🟡 架构设计完成，未部署 |
| **Pilot 用户** | Path B 招募中 | ≥5 个真实老年用户 | 🔴 招募阻塞 (V 未执行) |

**MVP 里程碑**:
1. ✅ Narrative Scorer 规则引擎完成
2. ✅ arXiv 论文草稿完成
3. ✅ 筛查问卷设计完成
4. ⏳ ASR 接入 (等 API Key)
5. ⏳ 最简 UI (微信 H5 或问卷星 + 手动流程)
6. ⏳ 首批用户招募 (Path B)
7. ⏳ 首个完整用户循环

**MVP 最小可行路径** (绕过当前阻塞):
```
方案 A: 全自动 (理想)
用户 → 微信小程序 → 语音录入 → ASR → Scorer → 报告

方案 B: 半自动 (现实可行)
用户 → 微信语音消息 → V 手动转发 → 讯飞听见转写 → Scorer → V 发送报告

方案 C: 纯手动 (最快验证核心假设)
用户 → 面对面录音 → 手动转写 → Scorer → 打印报告
```

**推荐**: 方案 B 或 C 先跑通，验证核心假设后再投入工程。

---

### 🟡 v1.0 — "可复制的服务流程"

**目标**: 10-50 个用户完成完整流程，服务可由非技术人员操作  
**预计时间**: 2026 Q3 (7-9 月)  
**前置条件**: MVP 核心假设验证通过  
**核心指标**: NPS ≥ 40, 完成率 ≥ 70%, 单用户服务时间 < 2h

| 组件 | v1.0 需要 | 备注 |
|------|-----------|------|
| **Narrative Scorer v1.0** | LLM 增强评分 (DashScope/GPT-4o) | 规则引擎 + LLM 仲裁 |
| **ASR** | 多方言支持 (普通话 + 粤语 + 吴语) | 讯飞/阿里 API |
| **CittaVoice v0.1** | 微信小程序 (语音采集 + 引导对话) | PTT 交互 |
| **报告生成** | 自动 PDF 报告 + 可选实体书 | CittaPress 基础版 |
| **SOP 手册** | 传记师操作手册 v1.0 | B2B2C 模式 |
| **数据安全** | 端到端加密 + 个人信息保护法合规 | 必须 |
| **定价验证** | 2999 RMB 基础版可行性 | 5-10 个付费用户 |

**v1.0 关键假设待验证**:
1. 老年用户能否接受语音交互方式？(MVP 验证)
2. 评分结果对临床有参考价值吗？(Pilot RCT 验证)
3. 2999 RMB 定价有付费意愿吗？(Midas 验证)
4. 传记师是否愿意使用 AI 工具？(B2B 验证)

---

### 🔵 v2.0 — "可规模化的平台"

**目标**: 100+ 用户/月，传记师生态初步形成  
**预计时间**: 2027 Q1 (最早)  
**前置条件**: v1.0 证明 PMF + 付费意愿  
**核心指标**: MRR > 10 万, 传记师 > 10 人, 用户留存 > 60%

| 组件 | v2.0 需要 | 备注 |
|------|-----------|------|
| **Narrative Scorer v2.0** | 多模态 (语音情感 + 面部表情 + 文本) | 需要 GPU |
| **CittaVoice v1.0** | 独立 App (iOS + 微信小程序) | 含视频通话 |
| **传记师平台** | SaaS 管理后台 (客户管理 + 订单 + 素材库) | B2B |
| **CittaPress v1.0** | 自动排版 + 印刷对接 + 物流 | 从手动到半自动 |
| **认知健康报告** | 纵向追踪 + 认知变化趋势图 | 需要 Pilot 数据 |
| **API 开放** | 第三方集成 (养老机构 EHR 对接) | B2B |
| **学术背书** | JMIR Aging / CHI 发表 | arXiv 升级 |
