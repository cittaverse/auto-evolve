## 变更概述

基于消融实验 V4 验证结果，L0 v0.6 移除了两个无贡献组件 (C2/C6)，并调整了仲裁阈值 (C7) 以降低过高的触发率。

### 消融实验 V4 关键发现

| 组件 | 验证结果 | 决策 |
|------|---------|------|
| C1: 情绪唤醒度检测 | 小幅增益 | ✅ 保留 |
| **C2: 动态理想比例** | **Mock 下无贡献** | ❌ **移除** |
| C3: LLM 事件提取 | 小幅增益 | ✅ 保留 |
| C4: L0 六维评分 | 最高影响力 | ✅ 保留 |
| C5: 多 Agent 评委 | 小幅增益 | ✅ 保留 |
| **C6: 投票加权** | **Mock 下无贡献** | ❌ **移除** |
| C7: 仲裁机制 | 显著影响 (需调整阈值) | ⚠️ **阈值上调** |

## 具体变更

### 1. C2 移除：动态理想比例 (enable_dynamic_ratio)

**变更内容**:
- 默认值从 `True` 改为 `False`
- 系统现在使用固定 60/40 比例，不再基于情绪唤醒度动态计算

**影响文件**:
- `pipeline/ablation_study.py`: 默认配置更新
- `pipeline/ablation_study.py`: `full_v0.6` 配置启用 `enable_dynamic_ratio=False`

**预期收益**:
- 计算成本降低 ~5%
- 评分逻辑简化，可解释性提升

---

### 2. C6 移除：投票加权 (enable_voting)

**变更内容**:
- 从置信度加权平均改为简单算术平均
- `weighted_average()` 方法重命名为 `simple_average()`

**影响文件**:
- `pipeline/judge_agent.py`: `simple_average()` 方法 (原 `weighted_average`)
- `pipeline/ablation_study.py`: 默认配置 `enable_voting=False`

**代码变更**:
```python
# v0.5 (加权平均)
weight = self.config['weights'].get(dim, 0.25) * result.confidence
weighted_sum += result.score * weight

# v0.6 (简单平均)
scores = [r.score for r in results.values()]
avg_score = sum(scores) / len(scores)
```

**预期收益**:
- 计算成本降低 ~5%
- 评分逻辑简化，避免置信度估计偏差

---

### 3. C7 调整：仲裁阈值上调 (conflict_threshold)

**变更内容**:
- 仲裁触发阈值从 **15 分** 上调至 **20 分**
- 仲裁调整上限从 **15 分** 上调至 **20 分**

**影响文件**:
- `config/judge_config.json`: `"conflict_threshold": 20.0`
- `config/judge_config.json`: `"adjustment_limit": 20.0`
- `pipeline/judge_agent.py`: 默认配置 `'conflict_threshold': 20.0`
- `pipeline/arbitrator_agent.py`: 默认配置 `'adjustment_limit': 20.0`

**变更原因**:
- 原 15 分阈值导致 100% 仲裁触发率 (过高)
- 20 分阈值可减少不必要的仲裁调用，同时保持对显著分歧的检测

**预期收益**:
- 仲裁调用减少 ~40-60% (估算)
- API 成本降低 ~10-15%
- 评分延迟降低
